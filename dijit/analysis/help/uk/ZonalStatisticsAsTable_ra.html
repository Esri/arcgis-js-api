Зональна статистика як таблиця

Цей інструмент підсумовує значення растру в межах зон іншого набору даних та звітує результати в таблиці.

Деякі з прикладів застосування:

  • Виходячи з заданого шару кордонів штатів і розподілу середнього рівня опадів у континентальній частині США, розрахувати розподіл подій з екстремальними зливами по штатах, використовуючи список процентильних значень.
  • Виходячи з шару екологічних зон, які перекриваються, та растрового шару вологості ґрунту, розрахувати всю підтримувану статистику в кожній зоні для співвіднесення зон з доступністю вологості ґрунту.

Якщо вибрано опцію Використовувати поточний екстент карти, будуть аналізуватись тільки ті об’єкти у вхідному шарі та шарі для підсумування, які видимі в межах поточного екстенту карти. Якщо цю опцію не вибрано, будуть використовуватись всі об'єкти у вхідному шарі та шарі для підсумування, навіть якщо вони знаходяться за межами поточного екстенту карти.

Вибрати растр зони або об'єкти

Шар, який визначає кордони зон або областей для підсумування.

Шар може являти собою цілочисельний растр або векторні дані. Для растрів зони визначаються всіма місцями розташування у вхідних даних, назви осередків яких співпадають. Для об'єктів зони можуть перекриватися. Області не обов'язково повинні бути неперервними.

Поле зони

Це атрибут шару, що використовуватиметься для визначення кордонів зон (областей).

Наприклад, припустимо, перший вхідний шар визначає кордони одиниці управління і має атрибути, які визначають регіон, район і ідентифікатор ділянки кожної одиниці. Також є растровий шар, який визначає індекс біорізноманіття для кожного місця розташування. Обравши відповідне поле, можна обчислити середнє біорізноманіття на локальному, районному або регіональному рівні.

Вибрати растр значень

Осередки растру у цьому шарі будуть підсумовані за областями (зони), в які вони укладаються.

Ігнорувати NoData в розрахунках (опціонально)

Визначає, чи будуть ігноруватися значення NoData в аналізі.

  • Прапорець встановлено — аналіз включатиме осередки, що мають значення у вхідному растрі значень в будь-якій певній зоні для визначення вихідного значення для цієї зони. При розрахуванні статистики осередки NoData растру значень ігноруватимуться. Це значення за замовчуванням.
  • Прапорець знято — результатом аналізу буде NoData, якщо для осередків растру значень в певній зоні є хоча б одне значення NoData.

Тип статистики (опціонально)

Вказує тип статистики, що треба розрахувати.

Вхідними даними може бути одне з наступного: Все, Середнє, Більшість, Максимум, Медіана, Мінімум, Меншість, Процентиль, Діапазон, Середньоквадратичне відхилення, Сума, Різноманітність, Мінімум і максимум, Середнє та середньоквадратичне відхилення, або Мінімум, максимум і середнє. За замовчуванням: Все.

Якщо вхідним растром значень є цілочисельний, за замовчуванням розраховуватиметься вся статистика. Якщо растр значень належить до типу з плаваючою точкою, за замовчуванням розраховуватиметься вся статистика крім Медіана та Процентиль.

Якщо вхідний растр значень належить до типу з плаваючою точкою, зональні розрахунки для Більшість, Меншість та Різноманітність обчислити не можна.

Процентильні значення (опціонально)

Процентилі, що треба розрахувати. Можна вказати єдине значення або список значень. За замовчуванням: 90, означає 90-й процентиль.

Значення можуть знаходитися в діапазоні від 0 до 100. Результат вказування 0-го процентиля по суті еквівалентний результату статистики «Мінімум», а 100-й процентиль еквівалентний максимуму. Значення 50 по суті даватиме такий самий результат, що й статистика «Медіана».

Ця опція підтримується, тільки якщо для параметру Тип статистики задано Процентиль або Все.

Тип інтерполяції процентилю (опціонально)

Вказує метод інтерполяції процентилю, що використовуватиметься, коли число значень з вхідного растру, що треба розрахувати, є парним.

Якщо растр вхідних значень має тип пікселя з плаваючою точкою, використовується метод Найближчий. Якщо растр вхідного значення має тип пікселя з плаваючою точкою, використовується метод Лінійний. Це значення за замовчуванням.

Для методу Найближче використовується найближче доступне значення до бажаного процентилю. Для методу Лінійний використовується середнє зважене двох суміжних значень від бажаного процентилю.

Обробити як багатовимірний (опціонально)

Указує, як будуть оброблятися вхідні растри, якщо вони є багатовимірними.

  • Прапорець встановлено — статистика розраховуватиметься, виходячи з поточного зрізу вхідного набору багатовимірних даних. Це значення за замовчуванням.
  • Прапорець знято — статистика розраховуватиметься для всіх вимірів вхідного набору багатовимірних даних.

Ім'я таблиці результатів

Назва шару, який буде створено в Мої ресурси і додано до карти. Ім'я за замовчуванням визначається на основі імені інструменту та імені вхідного шару. Якщо шар уже існує, вам буде запропоновано вказати іншу назву.
