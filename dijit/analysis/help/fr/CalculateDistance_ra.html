Calculer la distance

Cet outil calcule la distance euclidienne, la direction et l’allocation d’une seule source ou d’un ensemble de sources.

Cet outil permet, par exemple, d’apporter une réponse aux questions suivantes :

  • À quelle distance se trouve la ville plus proche ?
  • À quelle distance se trouve la route la plus proche ?
  • À quelle distance se trouve la source d’eau la plus proche ?

Si l’option Utiliser l’étendue courante de la carte est sélectionnée, seules ces entités qui sont visibles dans la couche en entrée au sein de l’étendue courante de la carte seront analysées. Si l’option n’est pas sélectionnée, toutes les entités qui se trouvent dans la couche en entrée seront analysées, même si elles se trouvent en dehors de l’étendue courante de l’affichage cartographique.

Choisir la couche raster ou la couche d'entités vers laquelle calculer la distance.

Couche qui définit les sources vers lesquelles la distance sera calculée. La couche peut être un service d’imagerie ou un service d’entités.

Pour le service d’imagerie, le type en entrée peut être entier ou à virgule flottante.

Pour le service d’entités, l’entrée peut être de type ponctuel, linéaire ou surfacique.

Distance maximale à calculer

Distance de calcul maximale. Si, par exemple; vous fournissez une valeur de 10 000 mètres, la distance euclidienne est calculée à partir des emplacements source jusqu’à cette valeur (distance).

Si aucune valeur n’est fournie, la distance est calculée dans la limite de l’étendue spécifiée.

Les unités disponibles prises en charge sont les kilomètres, les mètres, les miles et les pieds.

Taille de cellule en sortie

Taille de cellule et unité des rasters en sortie.

Les unités peuvent être les kilomètres, les mètres, les miles ou les pieds.

Nom de la couche de résultat de distance

Le nom de la couche qui sera créée dans la page My Content (Mon contenu) et ajoutée à la carte. Le nom par défaut repose sur le nom de l'outil et sur le nom de la couche en entrée. Si la couche existe déjà, vous êtes invité à fournir un autre nom.

Vous pouvez spécifier le nom d’un dossier dans My Content (Mon contenu), dans lequel le résultat sera enregistré via la zone de liste déroulante Save result in (Enregistrer le résultat dans). Si vous avez les privilèges permettant de créer des couches tuilées et d’images dynamiques, vous pouvez spécifier le type de couche à générer en sortie à l’aide de la zone déroulante Save result as (Enregistrer le résultat sous).

Choisir un champ d’allocation

Champ dans la couche source en entrée qui contient les valeurs définissant chaque source. Il peut s’agir d’un champ de type entier ou chaîne du jeu de données source.

La valeur par défaut est Value. Si aucun service d’entités et aucun champ Value n’est disponible, le premier champ de type entier ou chaîne disponible sera utilisé. Si aucun champ valide n'existe, le champ ObjectID (par exemple, OID ou FID, selon le type de l'entité entrée) est utilisé.

Choisir une couche raster ou d’entités pour représenter les interruptions

Jeu de données qui définit les interruptions.

Pour un service d’imagerie, le type en entrée peut être entier ou à virgule flottante. Les cellules dotées d’une valeur (y compris zéro) seront traitées comme une interruption. Les cellules de valeur NoData ne seront pas traitées comme une interruption.

Pour le service d’entités, l’entrée peut être de type ponctuel, linéaire ou surfacique.

Méthode de distance

Détermine si la distance doit être calculée avec une méthode plane (Terre plate) ou géodésique (ellipsoïde).

  • Planar (Planaire) : le calcul est effectué sur une surface plane projetée à l’aide d’un système de coordonnées cartésiennes 2D. Il s'agit de la méthode par défaut.
  • Geodesic (Géodésique) : les distances sont calculées sur l’ellipsoïde. Les résultats ne changent donc pas quelle que soit la projection en entrée ou en sortie.
