Copier dans le stockage des données

Cet outil copie une couche en entrée dans un stockage des données. Les données sont copiées dans ArcGIS Data Store et seront stockées dans votre stockage des données relationnelles ou spatio-temporelles.

Vous pouvez par exemple copier des entités stockées dans un partage de fichiers Big Data dans un stockage des données relationnelles et spécifier que seules les entités au sein de l'étendue actuelle de la carte seront copiées. Ceci crée un service d'entités hébergé avec uniquement les entités qui se trouvaient au sein de l'étendue de la carte spécifiée.

Choisir la couche à copier

La couche ou table qui va être copiée.

En plus de choisir une couche de votre carte, vous pouvez sélectionner Parcourir les couches au bas de la liste déroulante pour parcourir votre contenu et rechercher une couche d'entités ou un jeu de données de partage de fichiers Big Data.

Vous pouvez utiliser toutes les entités ou effectuer une sélection (facultatif)

Vous avez également la possibilité d'utiliser le bouton de sélection pour identifier les emplacements à copier.

Choisir le site ArcGIS Data Store dans lequel effectuer la copie

Le stockage des données dans lequel la couche va être copiée. Ce paramètre remplacera la sélection dans les environnements d'analyse Les options sont les suivantes :

  • Spatio-temporels : il s'agit de la valeur par défaut.
  • Relationnels.

Nom de la couche de résultat

Il s’agit du nom de la couche qui sera sur dans votre page My Content (Mon contenu) et ajoutée à la carte. Le nom par défaut repose sur le nom de l'outil et sur le nom de la couche en entrée. Si la couche existe déjà, l'outil échoue.

La zone de liste déroulante Save result in (Enregistrer le résultat dans) vous permet de spécifier le nom d’un dossier dans My content (Mon contenu) où le résultat sera enregistré.
