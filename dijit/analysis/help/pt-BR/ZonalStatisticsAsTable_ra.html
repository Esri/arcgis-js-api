Estatística Zonal como Tabela

Esta ferramenta resume os valores de um raster dentro das zonas de outro conjunto de dados e relata os resultados em uma tabela.

Alguns aplicativos de exemplo incluem o seguinte:

  • Dada uma camada de limites estaduais e a distribuição da precipitação média sobre o território continental dos Estados Unidos, calcule a distribuição estadual de eventos de chuva extrema usando uma lista de valores percentuais.
  • Dada uma camada de zonas ecológicas sobrepostas e uma camada raster de umidade do solo, calcule todas as estatísticas com suporte dentro de cada zona para relacionar as zonas com a disponibilidade de umidade do solo.

Se Utilizar extensão de mapa atual estiver selecionada, somente aquelas feições na camada de entrada e camada a serem resumidas que são visíveis dentro da extensão de mapa atual serão analisadas. Se não selecionada, todas as feições em ambas a camada de entrada e a camada a ser resumida serão analisadas, mesmo que estiverem fora da extensão de mapa atual.

Escolher feições ou raster de zona

A camada que define os limites das zonas, ou áreas, que serão resumidas.

A camada pode ser um raster inteiro ou dados de feições. Para rasters, as zonas são definidas por todos os locais na entrada que tenham o mesmo valor de célula. Para feições, as zonas podem ser sobrepostas. As áreas não têm que ser contíguas.

Campo de Zona

Este é o atributo da camada que será usada para definir os limites das zonas (áreas).

Por exemplo, suponha que a primeira camada de entrada define os limites da unidade de gerenciamento, com atributos que definem a região, o distrito e o parcel ID de cada unidade. Você também tem uma camada raster definindo um índice de biodiversidade de cada local. Com o campo que você selecionar, você pode decidir calcular a biodiversidade média no local, distrito ou níveis regionais.

Escolher raster de valor

As células raster desta camada serão resumidas pelas áreas (zonas) que elas caem dentro.

Ignorar NoData nos cálculos (opcional)

Especifica se os valores de NoData são ignorados na análise.

  • Marcado—A análise incluirá as células que possuem um valor no raster do valor de entrada dentro de qualquer zona específica para determinar o valor de saída para essa zona. As células NoData no raster de valor serão ignoradas no cálculo da estatística. Este é o padrão.
  • Desmarcado—A análise resultará em NoData se houver algum valor NoData para as células do raster de valor dentro de uma zona específica.

Tipo de estatística (opcional)

Especifica o tipo de estatística a ser calculado.

A entrada pode ser um dos seguintes: Todos, Média, Maioria, Máximo, Média, Mínimo, Minoria, Percentil, Intervalo, Desvio padrão, Soma, Variedade, Mínimo e Máximo, Média e Desvio padrão, ou Mínimo, Máximo e Média. O padrão é Todos.

Se o raster do valor de entrada for inteiro, por padrão, todas as estatísticas serão calculadas. Se o raster de valor for do tipo ponto flutuante, por padrão, todas as estatísticas, exceto Mediana e Percentil, serão calculadas.

Se o raster de valor de entrada for do tipo ponto flutuante, os cálculos zonais para Maioria, Minoria e Variedade não podem ser calculados.

Valores de percentil (opcional)

Os percentis a serem calculados. Um único valor ou uma lista de valores podem ser especificados. O padrão é 90, indicando o 90º percentil.

Os valores podem variar de 0 até 100. O resultado da especificação do 0º percentil é essencialmente equivalente ao da estatística mínima e o 100º percentil é equivalente ao máximo. Um valor de 50 produzirá essencialmente o mesmo resultado que a estatística mediana.

Esta opção só é suportada se o parâmetro Tipo estatísticas estiver definido como Percentil ou Todos.

Tipo de interpolação de percentil (opcional)

Especifica o método de interpolação de percentil a ser usado quando o número de valores do raster de entrada a ser calculado é par.

Se o raster de valor de entrada for do tipo pixel inteiro, o método Mais próximo é usado. Se o raster de valor de entrada for do tipo pixel de ponto flutuante, então o método Linear será usado. Este é o padrão.

Para o método Mais próximo, é usado o valor disponível mais próximo do percentil desejado. Para o método Linear, é usada a média ponderada dos dois valores circundantes do percentil desejado.

Processar como multidimensional (opcional)

Especifica como os rasters de entrada serão processados se forem multidimensionais.

  • Marcado—As estatísticas serão calculadas a partir da parte atual do conjunto de dados multidimensional de entrada. Este é o padrão.
  • Desmarcado—As estatísticas serão calculadas para todas as dimensões do conjunto de dados multidimensional de entrada.

Nome da tabela de resultados

O nome da camada que será criada em Meu Conteúdo e adicionada no mapa. O nome padrão é baseado no nome da ferramenta e o nome da camada de entrada. Se a camada já existir, você será solicitado para fornecer outro nome.
