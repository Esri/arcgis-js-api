Features verbinden

Für den Task "Features verbinden" werden zwei Layer angegeben. Mit "Features verbinden" werden Attribute aus einem Feature mit denen eines anderen Features basierend auf räumlichen, zeitlichen und Attributbeziehungen oder einer Kombination aus diesen verbunden. Das Werkzeug ermittelt alle Eingabe-Features, die den angegebenen Verbindungsbedingungen entsprechen, und verbindet den zweiten Eingabe-Layer mit dem ersten. Sie können optional alle Features mit den übereinstimmenden Features verbinden oder die übereinstimmenden Features zusammenfassen.

Der Task "Features verbinden" kann auf Punkte, Linien, Flächen und Tabellen angewendet werden. Eine zeitliche Verbindung erfordert Eingabedaten mit zeitaktivierten Eigenschaften und eine räumliche Verbindung erfordert, dass Ihre Daten eine Geometrie aufweisen.

Ziel-Layer auswählen

Der Layer, der Attribute des Join-Layers an seine Tabelle anhängen wird.

Wenn das Verarbeitungskoordinatensystem nicht auf ein projiziertes Koordinatensystem festgelegt ist, werden Sie aufgefordert, es festzulegen, wenn Sie die Analyse ausführen.

Verbindungsvorgang auswählen

Legt fest, wie Verbindungen zwischen dem Ziel- und dem Join-Layer in der Ausgabe verarbeitet werden, wenn mehrere Verbindungs-Features dieselbe Beziehung mit dem Layer, mit dem sie verbunden sind, aufweisen. Es stehen zwei Verbindungsvorgänge zur Auswahl:

  • Eins zu eins verbinden: Mit dieser Option werden alle übereinstimmenden Features mit jedem zu verbindenden Feature zusammengefasst.
  • Eins zu vielen verbinden: Mit dieser Option werden alle übereinstimmenden Features mit dem Join-Layer verbunden.

Angenommen, wir suchen Supermärkte im Umkreis von 2 Kilometern von einem Bauernmarkt. In diesem Fall muss der zu verbindende Layer über ein einzelnes Feature verfügen, das einen Bauernmarkt darstellt, und die Verbindungs-Features stellen die lokalen Lebensmittelgeschäfte mit Attributen wie dem jährlichen Gesamtumsatz dar. Mit dem Werkzeug Features verbinden wird ermittelt, dass fünf Lebensmittelgeschäfte diesen Kriterien entsprechen. Wenn der Verbindungsvorgang Eins zu viele verbinden festgelegt wird, enthält das Ergebnis fünf Features, wobei jede Zeile den Baumarkt und einen Supermarkt darstellt. Wird die Beziehung Eins zu eins verbinden festgelegt, führt dies zu einem Feature, das den Baumarkt und die zusammengefassten Informationen zu den Supermärkten darstellt, darunter die Anzahl (2) und andere Statistiken wie der Gesamtjahresumsatz.

Mindestens eine Verbindung auswählen

Sie können nur ein, zwei oder drei Verbindungstypen auswählen. Folgende Verbindungstypen sind verfügbar:

  • Räumlich: Verwendet eine festgelegte räumliche Beziehung zum Verbinden von Features. Dies erfordert, dass beide Layer eine Geometrie aufweisen.
  • Zeitlich: Verwendet eine zeitliche Beziehung zum Verbinden von Features. Dies erfordert, dass für beide Layer Zeiteigenschaften aktiviert werden.
  • Attribut: Verbindet Features basierend auf gleichen Feldern.

Räumliche Beziehung auswählen

Die räumliche Beziehung, anhand derer festgelegt wird, ob Features miteinander verbunden werden. Die verfügbaren Beziehungen hängen vom Geometrietyp (Punkt, Polylinie, Polygone) ab, der als Eingabe-Feature verwendet wird. Die folgenden Beziehungen sind verfügbar:

  • Schneidet: Features werden abgeglichen, wenn sie sich überschneiden.
  • Gleich: Die Features werden abgeglichen, wenn sie dieselbe Geometrie aufweisen.
  • Near: Die Features werden abgeglichen, wenn sich sich innerhalb einer angegebenen Entfernung voneinander befinden. Der Ziel-Layer muss sich in einem projizierten Koordinatensystem befinden, oder der Raumbezug für die Verarbeitung muss unter Analyseumgebungen auf ein projiziertes Koordinatensystem festgelegt werden, wenn die räumliche Beziehung Near lautet.
  • Enthält: Die Features werden abgeglichen, wenn der Layer, mit denen sie verbunden werden, in den Verbindungs-Features enthalten ist.
  • Innerhalb: Die Features werden abgeglichen, wenn der Layer, mit denen sie verbunden werden, innerhalb der Verbindungs-Features liegt.
  • Berührt: Die Features werden abgeglichen, wenn ihre Grenze das Feature berührt, mit dem sie verbunden werden.
  • Kreuzt: Die Features werden abgeglichen, wenn sie eine Umrisslinie mit Kreuzungen aufweisen.
  • Überlappt: Die Features werden verbunden, wenn sie sich überlappen.

Diese Entfernung gibt den Radius an, der auf eine räumliche Near-Beziehung angewendet wird.

Angenommen, Sie verfügen über ein Dataset, das ein Atomkraftwerk darstellt und ein Dataset, das Wohnhäuser darstellt. Sie können eine Entfernung von 1 Kilometer festlegen, um Häuser zu suchen, die sich im Umkreis von 1 Kilometer vom Atomkraftwerk befinden.

Zeitliche Beziehung auswählen

Die zeitliche Beziehung, anhand derer festgelegt wird, ob Features miteinander verbunden werden. Diese Option ist nur verfügbar, wenn für beide Layer Zeiteigenschaften aktiviert wurden; die verfügbaren Beziehungen hängen vom Zeittyp ab, der für die Eingabe-Features verwendet wird. Die folgenden Beziehungen sind verfügbar:

  • Entspricht: Das Feature wird abgeglichen, wenn das erste Feature dem zweiten entspricht.
  • Erfüllt von: Das Feature wird abgeglichen, wenn die Bedingungen der ersten Features vom zweiten erbracht werden.
  • Überlappt: Das Feature wird abgeglichen, wenn die ersten Features das zweite überlappen.
  • Überlappt von: Das Feature wird abgeglichen, wenn die ersten Features vom zweiten überlappt werden.
  • Während: Das Feature wird abgeglichen, wenn die ersten Features während des zweiten stattfinden.
  • Enthält: Das Feature wird abgeglichen, wenn das erste Feature das zweite enthält.
  • Gleich: Das Feature wird abgeglichen, wenn das erste Feature dem zweiten entspricht.
  • Beendet: Das Feature wird abgeglichen, wenn das erste Feature das zweite beendet.
  • Beendet von: Das Feature wird abgeglichen, wenn die ersten Features vom zweiten beendet werden.
  • Startet: Das Feature wird abgeglichen, wenn das erste Feature das zweite startet.
  • Gestartet von: Das Feature wird abgeglichen, wenn die ersten Features vom zweiten gestartet werden.
  • Schneidet: Die Features werden abgeglichen, wenn sie von allen Zeiten geschnitten werden.
  • Near: Die Features werden verbunden, wenn sie einer angegebenen Zeit entsprechend nahe beieinander liegen.

Diese zeitliche Entfernung gibt den zeitlichen Radius an, der auf eine zeitliche Near-Beziehung angewendet wird.

Angenommen, Sie verfügen über einen Layer mit Schiffsereignissen und einen Layer mit GPS-Spuren für einen Hurrikan. Sie können nach Schiffsereignissen innerhalb einer angegebenen räumlichen Entfernung (1 Kilometer) und einer zeitlichen Entfernung (5 Stunden) von Hurrikan-Spuren suchen. Dies würde dazu führen, dass Schiffsereignisse mit Hurrikanen verbunden werden, die räumlich und zeitlich nah beieinander lagen.

Zu übereinstimmende Felder auswählen

Diese Beziehung gleicht die Werte im Feld eines Layers mit den Werten im Feld eines anderen Layers ab.

Angenommen, es liegt ein landesweiter geographischer Layer mit Wohnadressen (mit einem ZIP-Feld) und ein Tabellen-Dataset mit demografischen Gesundheitsdaten nach PLZ-Code (ein Feld namens HEALTHZIP) vor. Das Gesundheits-Dataset lässt sich mit den Wohnadressendaten verbinden, indem das Feld ZIP mit dem Feld HEALTHZIP abgeglichen wird, was zu einem Layer mit Wohnsitzen führt, die den Gesundheitsdaten entsprechen.

Statistiken hinzufügen (optional)

Wenn der Verbindungsvorgang Eins zu eins verbinden lautet, können Sie Statistiken für Ihre verbundenen Features berechnen. Standardmäßig werden alle Statistiken berechnet.

Sie können Statistiken für Features berechnen, die zusammengefasst wurden. Für numerische Felder kann Folgendes berechnet werden:

  • Anzahl: Berechnet die Anzahl der Nicht-NULL-Werte. Kann für numerische Felder oder Zeichenfolgen verwendet werden. Die Anzahl von [NULL, 0, 2] beträgt 2.
  • Summe: Die Summe der numerischen Werte in einem Feld. Die Summe von [NULL, NULL, 3] beträgt 3.
  • Mittelwert: Der Mittelwert der numerischen Werte. Der Mittelwert von [0, 2, NULL] beträgt 1.
  • Min: Der Minimalwert eines numerischen Feldes. Der Minimalwert von [0, 2, NULL] beträgt 0.
  • Max: Der Maximalwert eines numerischen Feldes. Der Maximalwert von [0, 2, NULL] beträgt 2.
  • Bereich: Der Bereich eines numerischen Feldes. Dieser wird durch die Subtraktion der Minimalwerte vom Maximalwert berechnet. Der Bereich von [0, NULL, 1] lautet 1. Der Bereich von [NULL, 4] lautet 0.
  • Varianz: Die Varianz eines numerischen Feldes in einer Spur. Die Varianz von [1] beträgt Null. Die Varianz von [NULL, 1,1,1] beträgt Null.
  • Standardabweichung: Die Standardabweichung eines numerischen Feldes. Die Standardabweichung von [1] ist Null. Die Standardabweichung von [NULL, 1,1,1] ist 1.

Für Zeichenfolgenfelder kann Folgendes berechnet werden:

  • Anzahl: Die Anzahl der Nicht-NULL-Werte.
  • Beliebig: Diese Statistik ist eine auf dem Zufallsprinzip basierende Stichprobe eines Zeichenfolgenwertes im angegebenen Feld.
Alle Statistiken werden für Nicht-NULL-Werte berechnet. Der resultierende Layer enthält ein neues Feld für jede berechnete Statistik. Jede Anzahl von Statistiken kann durch Auswahl eines Attributs und einer Statistik ausgewählt werden.

Einen Ausdruck zum Verbinden von Features erstellen (optional)

Wendet eine Bedingung auf angegebene Felder an. Nur Features mit Feldern, die diese Bedingungen erfüllen, werden verbunden.

Ein Beispiel: Angenommen, Sie möchten eine Verbindung mit einem Dataset nur für die Features erstellen, für die health_spending (Gesundheitskosten) mehr als 20 Prozent vom income (Einkommen) ausmacht. Wenden Sie hierzu die Verbindungsbedingung $target["health_spending"] > ($join["income"] * .20) auf das Feld health_spending aus dem ersten Dataset (die Dataset-Features, mit denen die Verbindung erfolgt) und das Feld income aus dem zweiten Dataset (das Dataset, das verbunden wird) an.

Verbindungsbedingung können mithilfe des Ausdrucksrechners angewendet werden.

Data Store auswählen

GeoAnalytics-Ergebnisse werden in einem ArcGIS Data Store gespeichert und als Feature-Layer in Portal for ArcGIS dargestellt. In den meisten Fällen sollten Ergebnisse im Data Store vom Typ "spatiotemporal" gespeichert werden, was gleichzeitig die Standardeinstellung ist. Manchmal empfiehlt es sich auch, Ergebnisse im Data Store vom Typ "relational" zu speichern. Im Folgenden sind Gründe aufgeführt, die für ein Speichern der Ergebnisse in einem Data Store vom Typ "relational" sprechen:

  • Verwenden der Ergebnisse in portalübergreifender Zusammenarbeit
  • Aktivieren von Synchronisierungsfunktionen für Ihre Ergebnisse

Der Einsatz des Data Stores vom Typ "relational" ist nicht angezeigt, wenn abzusehen ist, dass Ihre GeoAnalytics-Ergebnisse weiter zunehmen. Verwenden Sie stattdessen die Funktionen des Big-Data-Store vom Typ "spatiotemporal", um große Datenmengen zu verwalten.

Name des Ergebnis-Layers

Dies ist der Name des Layers, der in Eigene Inhalte erstellt und der Karte hinzugefügt wird. Der Standardname basiert auf dem Werkzeugnamen und dem Namen des Eingabe-Layers. Wenn der Layer bereits vorhanden ist, kann das Werkzeug nicht ausgeführt werden.

Über das Dropdown-Feld Ergebnis speichern in können Sie den Namen eines Ordners in Eigene Inhalte angeben, in dem das Ergebnis gespeichert werden soll.
