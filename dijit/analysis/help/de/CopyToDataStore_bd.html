In Data Store kopieren

Dieses Werkzeug verwendet einen Eingabe-Layer und kopiert ihn in einen Data Store. Daten werden in den ArcGIS Data Store kopiert und im Data Store vom Typ "relational" oder im Data Store vom Typ "spatiotemporal" gespeichert.

Sie können beispielsweise Features, die in einer Big-Data-Dateifreigabe gespeichert sind, in einem Data Store vom Typ "relational" speichern und festlegen, dass nur Features innerhalb der aktuellen Kartenausdehnung kopiert werden sollen. Auf diese Weise wird ein gehosteter Feature-Service erstellt, der nur die Features enthält, die sich innerhalb der angegebenen Kartenausdehnung befinden.

Zu kopierenden Layer auswählen

Der Layer oder die Tabelle wird kopiert.

Sie können nicht nur einen Layer aus der Karte auswählen, sondern auch unten in der Dropdown-Liste Layer durchsuchen auswählen, um zu Ihren für ein Big-Data-Dateifreigabe-Dataset bzw. -Feature-Layer zu navigieren.

Sie können alle Features verwenden oder eine Auswahl vornehmen (optional)

Sie können auch die Auswahlschaltfläche verwenden, um die zu kopierenden Positionen zu ermitteln.

Den ArcGIS Data Store auswählen, in den kopiert werden soll

Der Data Store, in den der Layer kopiert wird. Dieser Parameter überschreibt die Auswahl in den Analyseumgebungen Folgende Optionen sind verfügbar:

  • Spatiotemporal: Dies ist die Standardeinstellung.
  • Relational.

Name des Ergebnis-Layers

Dies ist der Name des Layers, der in Eigene Inhalte erstellt und der Karte hinzugefügt wird. Der Standardname basiert auf dem Werkzeugnamen und dem Namen des Eingabe-Layers. Wenn der Layer bereits vorhanden ist, kann das Werkzeug nicht ausgeführt werden.

Über das Dropdown-Feld Ergebnis speichern in können Sie den Namen eines Ordners in Eigene Inhalte angeben, in dem das Ergebnis gespeichert werden soll.
