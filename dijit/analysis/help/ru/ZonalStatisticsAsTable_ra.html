Зональная статистика в таблицу

Этот инструмент суммирует значения растра в пределах зон другого набора данных и записывает результаты в таблицу.

Примерами применения могут быть:

  • По слою государственных границ и распределению среднего количества выпадающих осадков на континентальной части США рассчитать распределение экстремальных явлений по штатам, используя список значений в процентах.
  • По слою перекрывающихся экологических зон и растровому слою влажности почв рассчитать всю поддерживаемую статистику в каждой зоне, чтобы связать зоны с пригодностью влажности почв.

Если опция Использовать текущий экстент карты включена, будут анализироваться только те объекты входного слоя и слоя объединения, которые видны в момент анализа в текущем экстенте. Если опция отключена, будут анализироваться все объекты входного слоя и слоя объединения, даже если они находятся вне текущего экстента карты.

Выбор растра или объектов зон

Слой, задающий границы зон или областей, которые будут суммироваться.

Слой может быть целочисленным растром или векторными данными. Для растров все зоны задаются всеми местоположениями с одинаковым значением ячейки во входных данных. Для объектов зоны могут перекрываться. Области не обязательно должны быть непрерывными.

Поле зоны

Это атрибут слоя, который будет использоваться для задания границ зон (областей).

Представьте, к примеру, что первый слой содержит границы административных подразделений с атрибутами, задающими регион, район и ID участка каждого подразделения. У вас также есть растровый слой, определяющий индекс биоразнообразия для каждого места. Выбрав поле, вы можете вычислить среднее биоразнообразие на локальном, районном или региональном уровне.

Выбор растра значений

Ячейки растра данного слоя будут суммированы по областям (зонам), в которые они попадают.

Игнорировать значение NoData при вычислениях (дополнительно)

Определяет, будут ли игнорироваться значения NoData в анализе.

  • Отмечено – анализ будет включать ячейки, имеющие значение во входном растре в пределах любой определенной зоны, чтобы задавать выходное значение зоны. Ячейки NoData в растре значений будут игнорироваться в вычислении статистики. Это значение по умолчанию.
  • Не отмечено – Результатом анализа будет значение NoData, если есть какие-либо значения NoData для ячеек растра значений в пределах определенной зоны.

Тип статистики (дополнительно)

Задает вычисляемый тип статистики.

Входными данными может быть одно из следующего: Все, Среднее, Большинство, Максимум, Медиана, Минимум, Меньшинство, Процентиль, Диапазон, Среднеквадратическое отклонение, Сумма, Разнообразие, Минимум и Максимум, Среднее и Среднеквадратическое отклонение или Минимум, Максимум и Среднее. Значение по умолчанию – Все.

Если входной растр является целочисленным, по умолчанию будет вычислена вся статистика. Если входной растр типа с плавающей точкой, по умолчанию будет вычислена вся статистика, кроме Медиана и Процентиль.

Если входной растр значений типа с плавающей точкой, зональные вычисления Большинство, Меньшинство и Разнообразие производиться не будут.

Значения процентиля (дополнительно)

Вычисляемые процентили. Можно указать отдельное значение или список значений. Значение по умолчанию – 90, определяющее 90-й процентиль.

Значения могут варьироваться от 0 до 100. Результат указания 0-го процентиля, по существу, эквивалентен минимуму в статистике, а 100-го - максимуму. Значение 50 даст по существу тот же результат, что и медиана в статистике.

Эта опция поддерживается, если только параметр Тип статистики задан как Процентиль или Все.

Тип интерполяции процентиля (дополнительно)

Определяет метод интерполяции процентиля, который будет использоваться, когда количество значений входного растра, которое нужно вычислить, является четным.

Если входной растр значений - целочисленный, используется метод Ближайший. Если тип пикселов входного растра - значение с плавающей точкой, используется Линейный метод. Используется по умолчанию.

Для метода Ближайший используется ближайшее доступное значение к желаемому процентилю. Для метода Линейный - используется средневзвешенное значение двух окружающих значений из желаемого процентиля.

Обработать как многомерный (дополнительно)

Определяет, как будут обрабатываться входные растры, если они являются многомерными.

  • Отмечено – статистика будет вычисляться из текущего среза входного многомерного набора данных. Это значение по умолчанию.
  • Не отмечено – статистика будет вычисляться для всех измерений входного многомерного набора данных.

Имя итоговой таблицы

Имя слоя, который будет добавлен в Мои ресурсы и на карту. Имя слоя по умолчанию зависит от имени инструмента и имени входного слоя. Если имя слоя уже используется, появится запрос ввести новое имя.
