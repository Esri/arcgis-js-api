Агрегировать многомерный растр

Генерирует набор многомерных растровых данных путем комбинирования существующих переменных многомерного растра вдоль измерения.

Если у вас есть данные за 30 лет о температуре воды на поверхности моря и через каждые 5 метров на глубину до 100 метров, собираемые ежемесячно, вот лишь некоторые примеры использования:

  • Агрегировать ежемесячные данные температур в годовые.
  • Агрегировать ежемесячные данные температур в 4-месячные интервалы.
  • Агрегировать данные температур на глубинах от 0 до 25 метров, затем от 25 до 50 метров, затем от 50 до 100 метров.

Если отмечена опция Использовать текущий экстент карты, будут проанализированы только переменные слоя, видимые в текущем экстенте карты. Если опция отключена, будут анализироваться переменные всего слоя, даже если они находятся за пределами текущего экстента карты.

Выбрать слой многомерных изображений для агрегирования

Слой входных изображений многомерного растра, который будет агрегирован.

Выбрать измерение, вдоль которого агрегировать

Измерение агрегации. Это измерение, по которому переменные будут агрегированы.

Выбрать переменные для агрегирования

Переменная или переменные, которые будут агрегированы по заданному измерению. Если не указано конкретных переменных, будут агрегированы все переменные в пределах заданного измерения.

Например, для агрегирования суточных температурных данных в среднемесячные, необходимо указать температуру в качестве переменной для агрегирования. Если вы не указали ни одной переменной, а у вас есть суточные данные по температуре и влажности, обе переменные будут агрегированы в среднемесячные и выходной многомерный растр будет включать обе переменные.

Выбрать метод агрегирования

Определяет математический метод, который будет использован для комбинации агрегированных срезов в пределах интервала.

  • Среднее – Вычисляет среднее значение пиксела по всем срезам в интервале. Используется по умолчанию.
  • Максимум – Вычисляет максимальное значение пиксела по всем срезам в интервале.
  • Большинство – Вычисляет значение пиксела, которое чаще всего встречается по всем срезам в интервале.
  • Минимум – Вычисляет минимальное значение пиксела по всем срезам в интервале.
  • Меньшинство – Вычисляет значение пиксела, которое реже всего встречается по всем срезам в интервале.
  • Медианное – Вычисляет медианное значение пиксела по всем срезам в интервале.
  • Диапазон – Вычисляет диапазон значений пикселов по всем срезам в интервале.
  • средне-среднеквадратическое отклонение – Вычисляет средне-среднеквадратическое отклонение значений пиксела по всем срезам в интервале.
  • Сумма – Вычисляет сумму значений пикселов по всем срезам в интервале.
  • Множество – Вычисляет число уникальных значений пикселов по всем срезам в интервале.
  • Пользовательское – Вычисляет значение пиксела на основе пользовательской растровой функции.

Выбрать шаблон растровых функций

Пользовательская растровая функция, которая будет использована для вычислений значений пикселов агрегированных растров. Этот параметр является элементом шаблона растровой функции на портале. Ввод этого параметра требуется, если параметр Определение агрегации установлен на Пользовательский.

Выбрать определение агрегирования

Определяет интервал измерения, по которому данные будут агрегированы.

  • Все - значения данных будут агрегированы по всем срезам. Используется по умолчанию.
  • Ключевое слово интервала - данные переменных будут агрегированы с использованием общеизвестного интервала.
  • Интервальное значение - данные переменных будут агрегированы с использованием заданного пользователем интервала и единицы.
  • Интервальные диапазоны - данные переменных будут агрегированы в пределах указанных пар значений или дат.

Выбрать интервал ключевого слова

Указывает ключевое слово интервала, которое будет использовано при агрегировании по измерению. Это обязательный параметр, если Определение агрегации установлено на Ключевое слово интервала, и агрегация выполняется по времени.

  • Ежечасно - значения данных будут агрегироваться по временным шагам в один час, результаты будут содержать каждый час во временном ряду.
  • Ежедневно - значения данных будут агрегироваться по временным шагам в один день, результаты будут содержать каждый день во временных рядах.
  • Еженедельно - значения данных будут агрегироваться по временным шагам в одну неделю, результаты будут содержать каждую неделю во временных рядах.
  • Ежемесячно - значения данных будут агрегироваться по временным шагам в один месяц, результаты будут содержать каждый месяц во временных рядах.
  • Ежевартально - значения данных будут агрегироваться по временным шагам в один квартал, результаты будут содержать каждый квартал во временных рядах.
  • Ежегодно - значения данных будут агрегироваться по временным шагам в один год, результаты будут содержать каждый год во временных рядах.
  • Повтор ежедневно - значения данных будут агрегироваться по временным шагам в один день, результаты будут содержать одно агрегированное значение на юлианские сутки. Результат будет включать не более 366 дневных временных срезов.
  • Повтор еженедельно - значения данных будут агрегироваться по временным шагам в одну неделю, результаты будут содержать одно агрегированное значение на неделю. Выходные данные будут включать не более 53 недельных временных срезов.
  • Повтор ежемесячно - значения данных будут агрегироваться по временным шагам в один месяц, результаты будут содержать одно агрегированное значение на месяц. Выходные данные будут включать максимум 12 ежемесячных временных срезов.
  • Повтор ежеквартально - значения данных будут агрегироваться по временным шагам в один квартал, результаты будут содержать одно агрегированное значение на квартал. Выходные данные будут включать не более 4 квартальных временных срезов.
  • Каждую декаду - значения данных будут агрегироваться в 3 периода по 10 дней каждый. Последний период может содержать более или менее 10 дней. Выходные данные будут включать 3 среза для каждого месяца.
  • Каждые пять дней - значения данных будут агрегироваться в 6 периодов по 5 дней каждый. Последний период может содержать более или менее 5 дней. Выходные данные будут включать 6 среза для каждого месяца.

Интервал по значению

Размер интервала, который будет использован для агрегации. Этот параметр необходим, если параметр Определение агрегации установлен на Значение интервала.

Например, чтоб агрегировать данные ежемесячных температур за 30 лет с инкрементом в 5 лет, введите 5 в качестве Интервал значений и укажите для Единиц измерения – Годы.

Единица

Единицы измерения, которые будут применены к интервалу значений. Этот параметр необходим, если параметр Измерение является полем времени, а параметр Определение агрегации установлен на Значение интервала.

Если вы агрегируете по какому-либо другому измерению, а не по времени, эта опция будет недоступна и единицы для значения интервала будут соответствовать единицам измерения переменной входных многомерных растровых данных.

  • Часы - значения данных будут агрегированы по часовым временным срезам с заданным интервалом.
  • Дни - значения данных будут агрегированы по суточным временным срезам с заданным интервалом.
  • Недели - значения данных будут агрегированы еженедельным временным срезам с заданным интервалом.
  • Месяцы - значения данных будут агрегированы по ежемесячным временным срезам с заданным интервалом.
  • Годы - значения данных будут агрегированы по ежегодным временным срезам с заданным интервалом.

Задать диапазоны

Интервальные диапазоны будут использованы для агрегации групп значений. Они состоят из пар минимальных и максимальных значений с типом поля Double или Date. Этот параметр необходим, если параметр Определение агрегации установлен на Интервальные диапазоны.

Игнорировать пропущенные значения при вычислениях

Определяет, будут ли игнорироваться пропущенные значения в анализе.

  • Отмечено – анализ будет включать все действительные пикселы по данному измерению и игнорировать любые пикселы NoData. Используется по умолчанию.
  • Не отмечено – Результатом анализа будет значение NoData, если для этого пиксела по данному измерению есть хотя бы одно значение NoData.

Имя слоя результата

Имя слоя, который будет добавлен в Мои ресурсы и на карту. Имя слоя по умолчанию зависит от имени инструмента и имени входного слоя. Если имя слоя уже используется, появится запрос ввести новое имя.

Вы можете указать имя папки в Моих ресурсах, где будет сохранен результат, с помощью ниспадающего списка Сохранить результат в.
