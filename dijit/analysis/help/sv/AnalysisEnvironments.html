Miljöinställningar för portalanalys

Miljöinställningar för analyser i Portal for ArcGIS och ArcGIS Online.

Skriv över resultatlager

Alternativ för att skriva över befintliga lager. Om det eventuellt finns ett befintligt lager med samma namn i innehållet när det här alternativet är markerat skrivs det befintliga lagret över.

  • Markerat – Om det redan finns ett lager med samma namn i ditt innehåll kommer det att skrivas över. Utdata kommer att få samma koordinatsystem som indata.
  • Avmarkerat – Om namnet som anges för resultatlagret redan finns, inträffar ett fel och resultatlagret måste få ett nytt namn innan verktyget kan köras. Det här är standardinställningen.

Stäng analyswidgeten efter att analysen körts

Alternativ för att behålla analysdialogrutan öppen och aktiverad medan verktyget körs. Den här inställningen gör att du enkelt kan ändra analysparametrarna samtidigt som det föregående analysjobbet fortsätter att köras. Analysen är en process som består av flera steg. Genom att hålla verktygsdialogrutan öppen kan du enkelt ändra parametrarna och köra analysen igen.

  • Markerat – Analysverktygsfönstret stängs och fokus återgår till innehållsförteckningen. Det här är standardinställningen
  • Avmarkerat – Analysverktygsfönstret förblir öppet och aktiverat.

Lagra resultaten av analysen som driftade geoobjektlager

Alternativ för att ange om resultatet som lagret skapade av analysen ska vara en geoobjektsamling eller ett värdbaserat geoobjektlager.

  • Markerat – Resultaten av din analys används för att skapa ett geoobjektlager och ett objekt skapas i ditt innehåll. Det här är standardinställningen.
  • Avmarkerat – Resultaten av din analys används för att skapa en geoobjektsamling.

Utdatakoordinatsystem

Miljöinställning för rasteranalys som används för att definiera koordinatsystemet för resultatlagret.

  • Samma som indata – Analysresultatet använder samma koordinatsystem som indata. Det här är standardinställningen.
  • Enligt vad som anges – Välj koordinatsystem för analysresultatet. När det här alternativet har valts kan du klicka på jordglobsikonen för att välja alternativ i en lista med kända koordinatsystem eller lägga till den geografiska referensen WKID i den angivna rutan.
  • Lager <namn> – Välj ett befintligt lager i din webbkarta som ska användas för att avgöra koordinatsystemet för analysresultatet.

Bearbetande koordinatsystem

Miljöinställning för GeoAnalytics som används för att fastställa det koordinatsystem som analysen ska använda under körningen.

För en del analyser i GeoAnalytics krävs att bearbetningen körs i ett projicerat koordinatsystem. Alla verktyg som har implementerat gruppering av data eller linjära enheter kräver ett projicerat koordinatsystem.

  • Samma som indata – Analysen bearbetas i samma koordinatsystem som indata. Det här är standardinställningen.
  • Enligt vad som anges – Välj koordinatsystem för bearbetningen av analysen. När det här alternativet har valts kan du klicka på jordglobsikonen för att välja ett alternativ i en lista med kända projicerade koordinatsystem eller lägga till den geografiska referensen WKID i den angivna rutan.
  • Lager <namn> – Välj ett befintligt lager från webbkartan. Koordinatsystemet för det lager som valts kommer att användas för att fastställa vilket koordinatsystem som ska användas för att bearbeta analysen.

Utbredning

Miljöinställning som används för att fastställa den utbredning eller de gränser som ska användas när analysen körs. Alla indatageoobjekt som ligger helt inom eller korsar den angivna utbredningen kommer att användas i analysen.

  • Standard – Använd den utbredning som anges av verktyget.
  • Enligt vad som anges – Ange koordinaterna för att definiera utbredningen manuellt.
  • Lager <namn> – Välj ett befintligt lager från webbkartan. Den geografiska utbredningen för det lager som väljs kommer att användas för att fastställa den utbredning som används för att bearbeta analysen.

Snappa raster

När du utför en rasteranalys och ställer in miljön Snappa raster justeras utbredningen av utdatarasterlagret så att det matchar celljusteringen för det angivna Snappa raster-lagret.

Cellstorlek

När du utför en rasteranalys anger den här miljön cellstorlek eller upplösning som ska användas för att skapa utdatarasterlagret. Standardutdataupplösningen avgörs av den största cellstorleken för indatarasterlagret.

  • Minimum för indata – Använd den minsta cellstorleken för alla indatalager.
  • Maximum för indata – Använd den största cellstorleken för alla indatalager. Det här är standardinställningen.
  • Enligt vad som anges – Ange ett numeriskt värde för inställning av cellstorleken. Om det här alternativet är valt blir standardvärdet 1.
  • Lager <namn> – Anger cellstorleken till det valda rasterlagret.

Maska

När du kör en rasteranalys använder du miljön Maska för att ange ett lager som ska användas för att definiera ditt intresseområde. Det är endast de celler som hamnar inom analysmasken som används i analysåtgärden.

  • Masken kan vara antingen ett raster eller ett geoobjektlager.
  • Om analysmasken är ett raster kommer alla celler som har ett värde att användas för att definiera masken. Celler i ett maskraster som har värdet NoData anses ligga utanför masken och får värdet NoData i analysresultatlagret.
  • Om analysmasken är ett geoobjektlager omvandlas den internt till ett raster vid körningen. Därför bör du se till att och ställs in på lämpligt sätt för analysen.

ArcGIS Data Store

GeoAnalytic-resultat sparas i ett ArcGIS-datalager och visas som ett geoobjektlager i Portal for ArcGIS. I de flesta fall bör resultaten sparas i ett rumstemporalt datalager. Det här är standardinställningen. Följande är anledningar till varför du kanske vill spara resultat i ett relationellt datalager:

  • för att använda resultaten i samarbete portal till portal
  • för att aktivera synkroniseringsfunktioner med dina resultat.

Du bör inte använda ett relationellt datalager om du förväntar dig att dina GeoAnalytics-resultat kommer att öka och vill använda det rumstemporala big data-lagrets funktioner till att hantera stora mängder data.
