Estatísticas de Zona como Tabela

Esta ferramenta resume os valores de um raster dentro das zonas de outro conjunto de dados e comunica os resultados numa tabela.

Alguns exemplos de aplicações incluem as seguintes:

  • Dada uma camada de limites estaduais e a distribuição da precipitação média sobre o território continental dos Estados Unidos, calcule a distribuição estadual de eventos de chuva extrema usando uma lista de valores percentuais.
  • Dada uma camada de zonas ecológicas sobrepostas e uma camada raster de humidade do solo, calcule todas as estatísticas suportadas dentro de cada zona para relacionar as zonas com a disponibilidade de humidade do solo.

Se Usar extensão de mapa atual for validado, apenas esses elementos na camada de entrada e da camada a ser resumida que são visíveis dentro do mapa de extensão atual serão analisados. Se não validado, todos os elementos tanto na camada de entrada e na camada a serem resumidos vão ser analisados, mesmo que se encontrem fora da extensão do mapa atual.

Selecionar raster ou elementos de zona

A camada que define os limites das zonas, ou áreas, que serão resumidas.

A camada pode ser de dados raster ou de elementos de número inteiro. Para rasters, as zonas são definidas com base em todas as localizações na entrada que possuem o mesmo valor de célula. Para elementos, as zonas podem ser sobrepostas. As áreas não têm de ser contíguas.

Campo de zona

Este é o atributo da camada que será utilizado para definir os limites das zona (áreas).

Por exemplo, suponha que a primeira camada de entrada define os limites de unidades de gestão, com atributos que definem a região, o distrito e a ID de parcela de cada unidade. Tem ainda uma camada raster que define um índice de biodiversidade para cada local. Com o campo que selecionar, pode decidir calcular a biodiversidade média a nível local, distrital ou regional.

Selecionar raster de valor

As células raster nesta camada serão resumidas pelas áreas (zonas) em que se incluem.

Ignorar NoData em cálculos (opcional)

Especifica se os valores em NoData são ou não ignorados na análise.

  • Selecionado — A análise incluirá as células que possuem um valor no raster de valor de entrada dentro de qualquer zona particular para determinar o valor de saída para aquela zona. As células NoData no raster de valor serão ignoradas no cálculo estatístico. Isto é por omissão.
  • Não Selecionado — A análise resultará em NoData se existirem valores NoData para as células do raster de valor dentro de uma zona particular.

Tipo de estatísticas (opcional)

Especifica o tipo estatístico a calcular.

A entrada pode ser uma das seguintes: Tudo, Média, Maioria, Máximo, Mediana, Mínimo, Minoria, Percentil, Intervalo, Desvio Padrão, Soma, Variedade, Mínimo e Máximo, Média e Desvio Padrão ou Mínimo, Máximo e Média. A predefinição é Tudo.

Se o raster de valor de entrada for um número inteiro, por predefinição, todas as estatísticas serão calculadas. Se o raster de valor for do tipo ponto flutuante, por predefinição, todas as estatísticas exceto Mediana e Percentil serão calculadas.

Se o raster de valor de entrada for do tipo ponto flutuante, os cálculos de zona para Maioria, Minoria e Variedade não podem ser calculados.

Valores de percentil (opcional)

Os percentis a calcular. Um único valor ou uma lista de valores podem ser especificados. A predefinição é 90, indicando o 90.º percentil.

O intervalo dos valores é de 0 a 100. O resultado da especificação do 0.º percentil é essencialmente equivalente à estatística mínimo e o 100.º percentil é equivalente a máximo. Um valor de 50 irá essencialmente produzir o mesmo resultado que a estatística mediana.

sta opção só é suportada se o parâmetro Tipo de Estatística estiver definido como Percentil ou Tudo.

Tipo de interpolação de percentil (opcional)

Especifica o método de interpolação de percentil a ser usado quando o número de valores do raster de entrada a ser calculado é par.

Se o raster de valor de entrada for do tipo píxel inteiro, o método Mais próximo é usado. Se o raster de valor de entrada for do tipo píxel de ponto flutuante, o método Linear é usado. Isto é por omissão.

Para o método Mais próximo, é usado o valor disponível mais próximo do percentil pretendido. Para o método Linear, é usada a média ponderada dos dois valores circundantes do percentil.

Processar como multidimensional (opcional)

Especifica como os rasters de entrada serão processados se forem multidimensionais.

  • Selecionado — as estatísticas serão calculadas a partir da secção atual do conjunto de dados multidimensional de entrada. Isto é por omissão.
  • Não Selecionado — as estatísticas serão calculadas para todas as dimensões do conjunto de dados multidimensional de entrada.

Nome da tabela de resultados

O nome da camada que será criada em O Meu Conteúdo e adicionada ao mapa. O nome padrão é baseado no nome da ferramenta e do nome da camada de entrada. Se a camada já existir, ser-lhe-á pedido que introduza outro nome.
