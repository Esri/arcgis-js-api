Acumulação de Distância

Esta ferramenta calcula a distância acumulada para cada célula até às origens, permitindo a distância em linha reta, distância de custo, distância de superfície verdadeira e fatores verticais e horizontais.

Exemplos de aplicações incluem responder a questões tais como as seguintes:

  • Qual é a distância até à cidade mais próxima?
  • Qual é a distância de custo até à estrada mais próxima?
  • Qual é a distância em torno das barreiras até à fonte de água mais próxima?

Se Utilizar a extensão atual do mapa estiver selecionado, apenas serão analisados elementos na camada de entrada que sejam visíveis na extensão atual do mapa . Se não estiver selecionado, serão analisados todos os elementos na camada de entrada, mesmo que estejam fora da extensão atual do mapa.

Selecionar raster ou elementos de origem

Uma camada de rasters ou elementos que identifica as origens para as quais as distâncias serão calculadas.

Se a entrada for um raster, a mesma tem de consistir em células com valores válidos (zero é um valor válido) para as origens e às células restantes tem de ser atribuído o valor NoData.

Se a entrada for uma camada de elementos, a mesma pode ser um ponto, uma linha ou um polígono.

Selecionar raster ou elementos de barreira (opcional)

O conjunto de dados que define as barreiras.

Para uma camada raster, o tipo de entrada pode ser inteiro ou flutuante. Todas as células com um valor (incluindo zero) serão tratadas como uma barreira. Todas as células NoData serão tratadas como uma barreira.

Para um serviço de elementos, a entrada pode ser ponto, linha ou polígono.

Selecionar raster de superfície (opcional)

Um raster define os valores de elevação em cada localização da célula.

Os valores são utilizados para calcular a distância real da superfície coberta ao passar entre células.

Selecionar raster de custos (opcional)

Um raster que define a impedância ou custo para que se mova de modo planimétrico através de cada célula.

O valor na localização de cada célula representa a distância de custo-por-unidade para movimentar através da célula. Cada localização de célula é multiplicada pela resolução de célula enquanto também compensa o movimento diagonal para obter os custos totais da passagem pela célula.

Os valores do raster de custo podem ser números inteiros ou de ponto flutuante, mas não podem ser negativos ou zero (não pode ter um custo negativo ou zero).

Selecionar raster vertical (opcional)

Um raster que define os valores de z para cada localização da célula.

Os valores são utilizados para calcular a inclinação utilizada para identificar o fator vertical incorrido durante o movimento de uma célula para outra.

Fator vertical (opcional)

O Fator vertical define a relação entre o fator de custo vertical (VF) e o ângulo de movimento relativo vertical (VRMA).

Existem vários fatores com modificadores que pode selecionar para identificar um gráfico de fator vertical definido. Os gráficos são utilizados para identificar o fator vertical utilizado no cálculo do custo total para mover-se para uma célula vizinha.

Nas descrições abaixo, o VF define a dificuldade vertical encontrada ao mover-se de uma célula para a próxima e o modificador do VRMA identifica o ângulo de inclinação entre a célula De origem e a célula Para origem.

  • Binário — Se o VRMA for superior ao ângulo de corte baixo e inferior ao ângulo de corte elevado, o VF está definido para o valor associado ao fator zero; caso contrário, é infinito.
  • Linear — O VF é uma função linear do VRMA.
  • Linear inverso — O VF é uma função linear inversa do VRMA.
  • Linear simétrico— O VF é uma função linear do VRMA no lado negativo ou positivo do VRMA e as duas funções lineares são simétricas no que respeita ao eixo VF (y).
  • Linear inverso simétrico— O VF é uma função linear inversa do VRMA no lado negativo ou positivo do VRMA e as duas funções lineares são simétricas no que respeita ao eixo VF (y).
  • Cos — O VF é a função baseada no cosseno do VRMA.
  • Sec— O VF é a função baseada no secante do VRMA.
  • Cos-Sec — O VF é a função baseada no cosseno do VRMA quando o VRMA é negativo e é a função baseada no secante do VRMA quando o VRMA é não negativo.
  • Sec-Cos — O VF é a função baseada no secante do VRMA quando o VRMA é negativo e é a função baseada no cosseno do VRMA quando o VRMA é não negativo.

Os modificadores para os parâmetros verticais são os seguintes:

  • Fator zero — O fator vertical utilizado quando o VRMA é zero. Este fator posiciona a interceção Y da função especificada. Por definição, o fator zero não é aplicável a qualquer uma das funções verticais trignométricas (Cos, Sec, Cos-Sec ou Sec-Cos). A interceção Y é definida por estas funções.
  • Ângulo de corte baixo — O ângulo do VRMA abaixo do qual o VF será definido para infinito.
  • Ângulo de corte elevado — O ângulo do VRMA acima do qual o VF será definido para infinito.
  • Inclinação — A inclinação da linha reta utilizada com os parâmetros Linear e Linear inverso. A inclinação é especificada como uma fração de subida em excesso (por exemplo, 45 por cento da inclinação é de 1/45, que é inserido como 0,02222).

Selecionar raster horizontal (opcional)

Um raster que define a direção horizontal em cada célula.

Os valores no raster têm de ser números inteiros de 0 a 360, sendo 0 graus a norte, ou na direção da parte superior do ecrã, e aumentando no sentido dos ponteiros do relógio. As áreas planas devem ter um valor de -1. Os valores em cada localização serão utilizados em conjunto com Fator horizontal para determinar o custo horizontal incorrido ao deslocar-se de uma célula para os seus vizinhos.

Fator horizontal (opcional)

O Fator horizontal define a relação entre o fator de custo horizontal e o ângulo de movimento relativo horizontal.

Existem vários fatores com modificadores que pode selecionar para identificar um gráfico de fator horizontal definido. Os gráficos são utilizados para identificar o fator horizontal utilizado no cálculo do custo total para mover-se para uma célula vizinha.

Nas descrições abaixo, o fator horizontal (HF) define a dificuldade horizontal encontrada ao mover-se de uma célula para a próxima e o ângulo de movimento relativo horizontal (HRMA) identifica o ângulo entre a direção horizontal de uma célula e a direção do movimento.

As definições e os parâmetros disponíveis destes HF são os seguintes:

  • Binário — Se o HRMA for inferior ao ângulo de corte, o HF é definido para o valor associado ao fator zero; caso contrário, é infinito.
  • Para a frente— Apenas é permitido movimento para a frente. O HRMA tem de ser superior ou igual a 0 e inferior a 90 (0 <= HRMA < 90). Se o HRMA for superior a 0 e inferior a 45 graus, o HF da célula é definido para o valor associado ao fator zero. Se o HRMA for superior ou igual a 45 graus é utilizado o valor de modificador do valor lateral. O HF para qualquer HRMA igual ou superior a 90 graus está definido para infinito.
  • Linear — O HF é uma função linear do HRMA.
  • Linear inverso — O HF é uma função linear inversa do HRMA.

Os modificadores para as palavras-chave horizontais são os seguintes:

  • Fator zero — O fator horizontal utilizado quando o HRMA é 0. Este fator posiciona a interceção Y para qualquer uma das funções do fator horizontal.
  • Ângulo de corte — O ângulo do HRMA para além do qual o HF será definido para infinito.
  • Inclinação — a inclinação da linha reta utilizada com as palavras-chave do fator horizontal Linear e Linear Inverso. A inclinação é especificada como uma fração de subida em excesso (por exemplo, 45 por cento da inclinação é de 1/45, que é inserido como 0,02222).
  • Valor lateral — O HF quando o HRMA é superior ou igual a 45 graus e inferior a 90 graus quando é especificada a palavra-chave do fator horizontal Para a frente.

Acumulação inicial

O custo cumulativo inicial para iniciar o cálculo do custo.

Permite a especificação do custo fixo associado a uma origem. Em vez de começar com um custo de zero, o algoritmo de custo começa com o valor definido pela Acumulação inicial.

Os valores têm de ser zero ou superiores. O padrão é 0.

Acumulação máxima

A acumulação máxima para o viajante para uma origem.

Os cálculos de custos continuam para cada origem até que a acumulação especificada seja alcançada.

Os valores têm de ser maiores que zero. A acumulação predefinida está na extremidade do raster de saída.

Multiplicador de custo

O multiplicador a aplicar aos valores de custo.

Permite controlar o modo de viagem ou a magnitude numa origem. Quanto maior for o multiplicador, maior será o custo de se mover através de cada célula.

Os valores têm de ser maiores que zero. O padrão é 1.

Direção de deslocação

Especifica a direção do viajante ao aplicar os fatores horizontal e vertical.

De origem — O fator horizontal e o fator vertical serão aplicados a começar na origem de entrada e viajam para as células não origem. Isto é por omissão.

Para origem — O fator horizontal e o fator vertical serão aplicados a começar em cada célula não origem e viajam de volta para a origem de entrada.

Especifique a palavra-chave De origem ou Para origem, que será aplicada a todas as origens, ou especifique um campo nos dados de origem que contenha as palavras-chave para identificar a direção de deslocação para cada origem. Esse campo tem de conter a string FROM_SOURCE ou TO_SOURCE.

Método de distância (opcional)

Especifica se é calculada a distância através de um método planar (terra plana) ou geodésico (elipsoide).

  • Planar — O cálculo da distância será efetuado numa superfície plana projetada utilizando um sistema de coordenadas cartesiano em 2D. Este é o método predefinido.
  • Geodésico — O cálculo da distância será efetuado no elipsóide. Portanto, independentemente da projeção da entrada ou saída, os resultados não se alteram.

Nome do raster de acumulação de distância resultante

O raster de acumulação de distância contém a distância acumulativa para cada célula a partir de, ou até à origem de menor custo.

O raster de saída é do tipo flutuante.

O nome da camada que será criada em O Meu Conteúdo e adicionada ao mapa. O nome padrão é baseado no nome da ferramenta e do nome da camada de entrada. Se a camada já existir, ser-lhe-á pedido que introduza outro nome.

É possível especificar o nome de uma pasta em O Meu Conteúdo, onde o resultado será guardado utilizando a opção Guardar Resultado em da caixa suspensa. Nome do raster de direção de retorno resultante (opcional)

O raster da direção de retorno contém a direção calculada em graus. A direção identifica a próxima célula ao longo do menor percurso de volta à origem mais próxima, evitando ao mesmo tempo as barreiras.

O intervalo de valores vai dos 0 graus até aos 360 graus, com o 0 reservado para as células de origem. A direção para Este (direita) é 90, e os valores aumentam no sentido dos ponteiros do relógio (180 é sul, 270 é oeste, e 360 é norte).

O raster de saída é do tipo flutuante.

O nome da camada que será criada em O Meu Conteúdo. É necessário fornecer um nome, que não exista presentemente, em O Meu Conteúdo.
