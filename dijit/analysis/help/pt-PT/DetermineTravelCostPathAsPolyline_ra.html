Determinar Percurso de Custo de Viagem Como Polilinha

Esta ferramenta calcula o percurso de polilinha de menor custo entre destinos e origens.

Como exemplos de aplicações, incluem-se as seguintes:

  • Encontre o melhor percurso para uma nova estrada em termos de custos de construção.
  • Identifique o percurso a tomar a partir de vários locais suburbanos (origens) até ao centro comercial mais próximo (destino).

Se a opção Utilizar a extensão atual do mapa se encontrar selecionada, apenas os locais nas camadas de entrada que são visíveis na extensão atual do mapa serão analisados. Se não selecionar a opção, serão analisados todos os locais em ambas as camadas de entrada, mesmo que se encontrem fora da extensão atual do mapa.

Selecionar camada raster ou de camada de origem a partir da qual determinar percursos

Um serviço de imagens ou serviço de elementos que identifica as células a partir das quais o percurso de menor custo é determinado para o destino de menor custo.

Caso a entrada seja um serviço de imagem, a entrada consiste de células que têm valores válidos (zero é um valor válido), e às células restantes tem de ser atribuído o valor NoData.

Selecionar a camada raster de custo de viagem

Um raster que define a impedância ou custo para que se mova de modo planimétrico através de cada célula.

O valor na localização de cada célula representa a distância de custo-por-unidade para movimentar através da célula. Cada localização de célula é multiplicada pela resolução de célula enquanto também compensa o movimento diagonal para obter os custos totais da passagem pela célula.

Os valores do raster de custo podem ser números inteiros ou de ponto flutuante, mas não podem ser negativos ou zero (não pode ter um custo negativo ou zero).

Raster ou Elementos de Destino de Entrada

Uma entrada que define os destinos utilizados para calcular a distância. A camada pode ser de dados ou de elementos.

Caso a entrada seja um serviço de imagem, a entrada consiste de células que têm valores válidos (zero é um valor válido), e às células restantes tem de ser atribuído o valor NoData.

Nome da camada de polilinhas resultante

O nome da camada que será criada em O Meu Conteúdo e adicionada ao mapa. O nome padrão é baseado no nome da ferramenta e do nome da camada de entrada. Se a camada já existir, ser-lhe-á pedido que introduza outro nome.

É possível especificar o nome de uma pasta em O Meu Conteúdo, onde o resultado será guardado utilizando a opção Guardar Resultado em da caixa suspensa. Se tiver os privilégios para criar camadas de imagens em mosaico e dinâmicas, pode especificar o tipo de camada que será gerado na saída utilizando a caixa pendente Guardar resultado como.

Selecionar tipo de percursos a determinar

Define o modo como os valores e zonas no destino de entrada serão interpretados nos cálculos de custo de percurso

  • Melhor Único — Para todas as células ou locais nos dados de destino de entrada, o percurso de menor custo é derivado do local com o mínimo dos percursos de menor custo até ao local de origem. Esta é a predefinição.
  • Para Cada Célula — Para cada célula ou local com valores válidos nos dados de destino de entrada, é determinado e guardado um percurso de menor custo no serviço de elementos de saída. Com esta opção, cada célula dos dados de destino de entrada é tratada separadamente, e é determinado um percurso de menor custo para cada célula.
  • Para Cada Zona — Para cada zona nos dados de destino de entrada, é determinado e guardado um percurso de menor custo no serviço de elementos de saída. Com esta opção, o caminho de menor custo para cada zona começa na célula ou local com a menor ponderação de custo e distância na zona.

Escolha o campo para obter valores para destino

O campo utilizado para obter valores para os locais de destino.
