Definições do Ambiente de Análise do Portal

Definições do ambiente de análise no Portal for ArcGIS e no ArcGIS Online.

Substituir camada resultante

Opção para substituir camadas existentes. Quando marcada, se já existir uma camada com o mesmo nome nos seus conteúdos, esta será substituída.

  • Marcada—Quando marcada, se uma camada com o mesmo nome já existir nos seus conteúdos, esta será substituída. A saída terá o mesmo sistema de coordenadas que a entrada.
  • Desmarcada—Caso o nome da camada de resultados introduzido já exista, ocorrerá um erro e o nome da camada de resultados tem de ser alterado antes de a ferramenta poder ser executada. Isto é por omissão.

Fechar widget de análise após executar a análise

Opção para manter a caixa de diálogo de análise aberta e ativa enquanto a ferramenta está em processamento. Esta definição irá permitir-lhe aplicar modificações, com facilidade, aos seus parâmetros de análise enquanto a tarefa de análise anterior continua em execução. A análise é um processo iterativo e, mantendo a caixa de diálogo aberta, permite-lhe efetuar alterações de parâmetros e voltar a executar a sua análise com facilidade.

  • Marcada—A janela da ferramenta de análise será encerrada e o foco dirigir-se-á novamente para o índice. Esta é a predefinição.
  • Desmarcada—A ferramenta de análise manter-se-á aberta e ativa.

Armazenar resultados de análise como camadas de elementos alojadas

Opção para especificar se a camada de resultado criada a partir da sua análise será uma coleção de elementos ou uma camada de elementos alojada.

  • Marcada—Os resultados da sua análise darão origem a uma camada de elementos alojada e será criado um item nos seus conteúdos. Isto é por omissão.
  • Desmarcada—Os resultados da sua análise darão origem a uma coleção de elementos.

Sistema de coordenadas de saída

Definição de ambiente para análise raster utilizada para definir o sistema de coordenadas da camada de resultados.

  • Igual à Entrada—O resultado da sua análise encontrar-se-á no mesmo sistema de coordenadas que o da entrada. Isto é por omissão.
  • Como especificado—Selecione o sistema de coordenadas para os resultados da sua análise. Quando esta opção se encontra selecionada, clique no ícone de globo para selecionar a partir de uma lista de sistemas de coordenadas conhecidos, ou adicione a referência espacial WKID na caixa.
  • <name> da camada—Selecione uma camada existente no seu mapa web, que será utilizada para determinar o sistema de coordenadas dos resultados da análise.

A processar sistema de coordenadas

As definições de ambiente para GeoAnalytics utilizadas para determinar o sistema de coordenadas que a sua análise utilizará durante a execução da análise.

No GeoAnalytics, algumas análises requerem que o processamento seja executado num sistema de coordenadas projetado. Qualquer ferramenta que tenha implementada a compartimentação (binning) de dados ou de unidades lineares necessita de um sistema de coordenadas projetado.

  • Igual à Entrada—A sua análise será processada no mesmo sistema de coordenadas que o da entrada. Isto é por omissão.
  • Como especificado—Selecione o sistema de coordenadas para para processar a sua análise. Quando esta opção se encontra selecionada, clique no ícone de globo para selecionar a partir de uma lista de sistemas de coordenadas projetadas conhecidos, ou adicione a referência espacial WKID na caixa.
  • <name> da camada—Selecione uma camada existente do seu mapa web. O sistema de coordenadas da camada escolhida será utilizado para determinar o sistema de coordenadas que será utilizado para processar a sua análise.

Extensão:

As definições de ambiente utilizadas para determinar a extensão ou o limite a utilizar durante a execução da análise. Todos os elementos de entrada que se encontram completamente incluídos ou que intersetam a extensão especificada serão utilizados na análise.

  • Padrão—Utilize a extensão fornecida pela ferramenta.
  • Como especificado—introduza as coordenadas manualmente para definir a extensão.
  • <name> da camada—Selecione uma camada existente do seu mapa web. A extensão espacial da camada escolhida será utilizada para determinar a extensão utilizada para processar a sua análise.

Encaixe de Raster

Ao efetuar análise raster, definir o ambiente Snap Raster irá ajustar a extensão da camada raster de saída por forma a que corresponda ao alinhamento de células da camada Snap Raster especificada.

Tamanho de Células

Ao efetuar análise raster, este ambiente define o tamanho ou resolução das células que serão utilizadas para criar a camada raster de saída. A resolução de saída predefinida é determinada a partir do maior tamanho de célula existente na camada raster de entrada.

  • Mínimo de Entradas—Utilize o tamanho de célula mais pequeno de todas as camadas de entrada.
  • Máximo de Entradas—Utilize o maior tamanho de célula de todas as camadas de entrada. Isto é por omissão.
  • Como especificado—Especifique um valor numérico para definir o tamanho das células. Caso esta opção seja selecionada, o valor predefinido é 1.
  • <name> da camada—Define o tamanho das células para a camada raster selecionada.

Máscara

Ao efetuar análise raster, utilize o ambiente Máscara para especificar uma camada que será utilizada para definir a sua área de interesse. Apenas as células que se enquadram na máscara de análise serão consideradas na operação de análise.

  • A máscara pode ser uma camada raster ou de elementos.
  • Caso a máscara da análise seja uma uma camada raster, todas as células que possuam um valor serão consideradas na definição da máscara. As células existentes numa máscara raster que sejam NoData serão consideradas como estando fora da máscara e não existirá qualquer elemento NoData na camada de resultados de análise.
  • Caso a máscara de análise seja uma camada de elementos, será internamente convertida para raster na execução. Por esta razão, garanta que o e são definidos de modo adequado à sua análise.

ArcGIS Data Store

Os resultados de GeoAnalytics são armazenados num ArcGIS Data Store e exibidos como uma camada de elementos no Portal for ArcGIS. Na maioria dos casos, os resultados deverão ser armazenados num armazenamento de dados espaciotemporal. Isto é por omissão. As seguintes razões justificam a opção pelo armazenamento de resultados num armazenamento de dados relacional:

  • Para utilizar os seus resultados em colaboração portal-to-portal
  • Para ativar funcionalidades de sincronização com os seus resultados

Não deverá utilizar um armazenamento de dados relacional caso antecipe um aumento dos seus resultados de GeoAnalytics, e caso pretenda tirar partido das funcionalidades de armazenamento de big data para gerir grandes quantidades de dados.
