Melhor Percurso Como Linha

Esta ferramenta calcula o percurso ideal dos destinos às origens como uma linha.

Como exemplos de aplicações, incluem-se as seguintes:

  • Encontre o melhor percurso para uma nova estrada em termos de custos de construção.
  • Encontre a melhor rota de navegação entre um destino e a localização atual.

Selecione o raster ou elementos de destino

Uma camada raster ou de elemento que identifica as células a partir das quais o percurso ideal é determinado até a origem ideal.

Se a entrada for um raster, a mesma tem de consistir em células com valores válidos (zero é um valor válido) para os destinos e às células restantes tem de ser atribuído o valor NoData. O tipo de raster de entrada tem de ser de tipo número inteiro.

Campo de destino

O campo utilizado para obter valores para os locais de destino.

Este campo tem de ser de tipo número inteiro.

Selecionar o raster de acumulação de distância

O raster de acumulação de distância é utilizado para determinar o melhor percurso desde os destinos até às origens.

O raster de acumulação de distância geralmente é criado com as ferramentas de Acumulação de Distância ou Atribuição de Distância. Cada célula no raster de acumulação de distância representa a distância mínima do custo acumulado sobre a superfície de cada célula para um conjunto de células de origem.

Selecionar raster de direção de retorno ou de direção de fluxo

O raster da direção de retorno contém as direções calculadas em graus. A direção identifica a próxima célula ao longo do melhor percurso de volta à origem de menor custo acumulado, evitando ao mesmo tempo as barreiras.

O raster de direção de retorno geralmente é criado com as ferramentas de Acumulação de Distância ou Atribuição de Distância. Neste caso, o intervalo de valores vai dos 0 graus até aos 360 graus. O valor 0 é reservado para as células de origem. A direção para Este (direita) é 90 graus, e os valores aumentam no sentido dos ponteiros do relógio (180 é sul, 270 é oeste, e 360 é norte).

O percurso ideal criado pode ser um percurso de fluxo com base na direção de fluxo D8. Para gerar um percurso ideal desta forma, use um raster de direção de fluxo D8 como entrada para este parâmetro.

Tipo de percurso (opcional)

Especifica uma palavra-chave que define como os valores e zonas nos dados de destino de entrada serão interpretados nos cálculos do percurso de custo.

  • Para Cada Zona — Para cada zona nos dados de destino de entrada, é determinado um percurso de menor custo. Com esta opção, o caminho de menor custo para cada zona começa na célula com a menor ponderação de distância de custo na zona. Isto é por omissão.
  • Melhor Único — Para todas as células nos dados de destino de entrada, o caminho de menor custo deriva da célula com o mínimo de caminhos de menor custo até às células de origem.
  • Para Cada Célula — Para cada célula com valores válidos nos dados de destino de entrada é determinado um percurso de menor custo. Com esta opção, cada célula dos dados de destino de entrada é tratada separadamente e é determinado um caminho de menor custo para cada célula.

Nome da camada de elementos de resultado

O nome da camada que será criada em O Meu Conteúdo e adicionada ao mapa. O nome padrão é baseado no nome da ferramenta e do nome da camada de entrada. Se a camada já existir, ser-lhe-á pedido que introduza outro nome.

É possível especificar o nome de uma pasta em O Meu Conteúdo, onde o resultado será guardado utilizando a opção Guardar Resultado em da caixa suspensa.
