ポータルでの解析の環境設定

Portal for ArcGIS および ArcGIS Online での解析の環境設定。

結果レイヤーを上書き

既存のレイヤーを上書きするオプション。オンにすると、同じ名前のレイヤーがコンテンツ内にすでに存在する場合は、そのレイヤーが上書きされます。

  • オン - 同じ名前のレイヤーがコンテンツ内にすでに存在する場合は、そのレイヤーが上書きされます。出力の座標系が入力と同じになります。
  • オフ - 指定された結果レイヤーの名前がすでに存在する場合は、エラーが発生します。ツールを実行する前に結果レイヤーの名前を変更しておく必要があります。これがデフォルト設定です。

分析の実行後に分析ウィジェットを閉じる

ツールの実行中に解析ダイアログを開いたまま有効な状態にしておくオプション。この設定を使用すると、前の解析ジョブの実行が継続中の場合でも、簡単に解析パラメーターを変更することができます。解析は反復的なプロセスであるため、ツール ダイアログを開いたままにしておくことで、簡単にパラメーターを変更し、解析を再実行できます。

  • オン - 解析ツールのウィンドウが閉じて、[コンテンツ] ウィンドウにフォーカスが戻されます。これがデフォルトです。
  • オフ - 解析ツールのウィンドウが開いたまま有効な状態になります。

解析結果をホスト フィーチャ レイヤーとして保存

解析から作成する結果レイヤーをフィーチャ コレクションにするかホスト フィーチャ レイヤーにするかを指定するオプション。

  • オン - 解析結果がホスト フィーチャ レイヤーとして作成され、アイテムがコンテンツ内に作成されます。これがデフォルト設定です。
  • オフ - 解析結果がフィーチャ コレクションとして作成されます。

出力座標系

結果レイヤーの座標系を定義する際に使用されるラスター解析の環境設定。

  • 入力データと同様 - 解析結果の座標系が入力と同じになります。これがデフォルト設定です。
  • 値指定 - 解析結果の座標系を選択します。このオプションをオンにした場合は、地球アイコンをクリックして既知の座標系のリストから座標系を選択するか、指定のボックスに空間参照の WKID を追加します。
  • レイヤー <名前> - Web マップ内の既存のレイヤーの中から、解析結果の座標系の決定に使用するレイヤーを選択します。

処理に使用する座標系

解析の実行中に使用される座標系を決定するための GeoAnalytics の環境設定。

GeoAnalytics での一部の解析では、処理を投影座標系で実行する必要があります。データまたは距離単位のラスター化を実装しているツールでは投影座標系が必要です。

  • 入力データと同様 - 解析が入力と同じ座標系で処理されます。これがデフォルト設定です。
  • 値指定 - 解析を処理する座標系を選択します。このオプションをオンにした場合は、地球アイコンをクリックして既知の投影座標系のリストから投影座標系を選択するか、指定のボックスに空間参照の WKID を追加します。
  • レイヤー <名前> - Web マップから既存のレイヤーを選択します。選択したレイヤーの座標系に基づいて、解析の処理に使用される座標系が決定されます。

Extent

解析の実行中に使用される範囲または境界を決定するための環境設定。指定の範囲内に完全に含まれているか、指定の範囲と交差しているすべての入力フィーチャが解析に使用されます。

  • デフォルト - このツールから提供される範囲が使用されます。
  • 値指定 - 座標系を手動で入力して範囲を定義します。
  • レイヤー <名前> - Web マップから既存のレイヤーを選択します。選択したレイヤーの空間範囲に基づいて、解析の処理に使用される範囲が決定されます。

スナップ対象ラスター

ラスター解析の実行時に、 [スナップ対象ラスター] 環境の設定に基づいて、指定した [スナップ対象ラスター] レイヤーのセルの配置と同じになるように出力ラスター レイヤーの範囲が調整されます。

セル サイズ

ラスター解析の実行時に、この環境に基づいて、出力ラスター レイヤーの作成に使用されるセル サイズまたは解像度が設定されます。デフォルトの出力解像度は、入力ラスター レイヤーの最も大きいセル サイズによって決まります。

  • 入力データの最小セルサイズ - すべての入力レイヤーの中から最小のセル サイズを使用します。
  • 入力データの最大セルサイズ - すべての入力レイヤーの中から最大のセル サイズを使用します。これがデフォルト設定です。
  • 値指定 - 数値を指定してセル サイズを設定します。このオプションを選択すると、デフォルト値が「1」に設定されます。
  • レイヤー <名前> - セル サイズを選択されたラスター レイヤーに設定します。

マスク

ラスター解析の実行時に、 [マスク] 環境を使用して、対象エリアの定義に使用されるレイヤーを指定します。解析マスク内にあるセルだけが解析処理で考慮されます。

  • マスクは、ラスターでもフィーチャ レイヤーでもかまいません。
  • 解析マスクがラスターの場合、値を持つすべてのセルがそのマスクを指定すると見なされます。マスク ラスター内の NoData のセルはマスクの外にあると見なされ、解析結果レイヤーでも NoData になります。
  • 解析マスクがフィーチャ レイヤーの場合は、実行時に内部でラスターに変換されます。そのため、解析に適した と が設定されていることを確認する必要があります。

ArcGIS Data Store

GeoAnalytic の結果は ArcGIS Data Store に保存され、フィーチャ レイヤーとして Portal for ArcGIS に公開されます。多くの場合、結果はビッグ データ ストアに保存されます。これがデフォルト設定です。結果をリレーショナル データ ストアに保存する理由は以下のとおりです。

  • 結果をポータル間での連携に使用するため。
  • 結果との同期機能を有効にするため

GeoAnalytics の結果が増加すると予想されており、大量のデータを処理するビッグ データ ストアの機能を利用したい場合には、リレーショナル データ ストアを使用すべきではありません。
