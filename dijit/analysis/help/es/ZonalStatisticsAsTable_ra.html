Estadísticas zonales como tabla

Esta herramienta resume los valores de un ráster dentro de las zonas de otro dataset y notifica los resultados en una tabla.

A continuación se indican algunas aplicaciones de ejemplo:

  • Dada una capa de fronteras entre estados y la distribución de precipitación media sobre la región continental de Estados Unidos, calcule la distribución estatal de eventos de lluvia extremos utilizando una lista de percentiles.
  • Dada una capa de zonas ecológicas superpuestas y una capa ráster de humedad del suelo, calcule todas las estadísticas compatibles en cada zona para relacionar zonas con la disponibilidad de humedad del suelo.

Si se ha activado Usar la extensión del mapa actual, solo se analizarán las entidades de la capa de entrada y la capa que se va a resumir que estén visibles en la extensión de mapa actual. Si la opción está desactivada, se analizarán todas las entidades de la capa de entrada y de la capa que se va a resumir, aunque estén fuera de la extensión de mapa actual.

Elegir entidades o ráster de zona

La capa que define los límites de las zonas, o áreas, que se van a resumir.

La capa puede ser un ráster entero o datos de entidad. Para los rásteres, las zonas se definen mediante todas las ubicaciones de la entrada que tienen el mismo valor de celda. En el caso de las entidades, las zonas se pueden superponer. No es necesario que las áreas sean contiguas.

Campo de zona

Este es el atributo de la capa que se utilizará para definir los límites de las zonas (áreas).

Por ejemplo, supongamos que la primera capa de entrada define los límites de la unidad de gestión, con atributos que definen la región, el distrito y el ID de parcela de cada unidad. También tiene una capa ráster que define el índice de biodiversidad para cada ubicación. Con el campo seleccionado, puede decidir calcular la biodiversidad media en el nivel local, de distrito o regional.

Elegir ráster de valor

Las celdas ráster de esta capa se resumirán por las áreas (zonas) en las que queden dentro.

Ignorar los NoData en los cálculos (opcional)

Especifica si los valores NoData se ignorarán en el análisis.

  • Activado: el análisis incluirá las celdas que tienen un valor en el ráster de valor de entrada dentro de una zona en particular para determinar el valor de salida de dicha zona. Las celdas NoData del ráster de valores se ignorarán en el cálculo estadístico. Esta es la opción predeterminada.
  • Desactivado: el análisis dará como resultado NoData si hay valores NoData para las celdas del ráster de valores dentro de una zona determinada.

Tipo de estadísticas (opcional)

Especifica el tipo de estadística que se calculará.

La entrada puede ser una de las siguientes: Todo, Media, Mayoría, Máximo, Mediana, Mínimo, Minoría, Percentil, Rango, Desviación estándar, Suma, Variedad, Mínimo y máximo, Media y desviación estándar o Mínimo, máximo y valor medio. El valor predeterminado es Todo.

Si el ráster de valor de entrada es entero, de forma predeterminada, se calcularán todas las estadísticas. Si el ráster de valor es de tipo coma flotante, de forma predeterminada, se calcularán todas las estadísticas con la excepción de Mediana y Percentil.

Si el ráster de valor de entada es de tipo coma flotante, los cálculos zonales para Mayoría, Minoría y Variedad no se pueden calcular.

Valores de percentil (opcional)

Percentiles para el cálculo. Se puede especificar un valor único o una lista de valores. El valor predeterminado es 90, que indica el 90.º percentil.

Los valores pueden variar de 0 a 100. El resultado de especificar el percentil 0.º es esencialmente equivalente al de la estadística mínima, y el percentil 100 es equivalente al máximo. Un valor de 50 generará en esencia el mismo resultado que la estadística mediana.

Esta opción solo está disponible si el parámetro Tipo de estadísticas está establecido en Percentil o Todo.

Tipo de interpolación de percentil (opcional)

Especifica el método de interpolación percentil que se va a utilizar si el número de valores del ráster de entrada calculado es par.

Si el ráster de valor de entrada es de tipo píxel entero, se utiliza el método Más cercano. Si el ráster de valor de entrada es de tipo píxel de punto flotante, se utiliza el método Lineal. Esta es la opción predeterminada.

Para el método Más cercano, se utiliza el valor disponible más cercano para el percentil deseado. Para el método Lineal, se utiliza la media ponderada de los dos valores circundantes del percentil deseado.

Procesar como multidimensional (opcional)

Especifica cómo se procesan los rásteres de entrada si son multidimensionales.

  • Activado: se calculan estadísticas de la parte actual de un dataset multidimensional. Esta es la opción predeterminada.
  • Desactivado: las estadísticas se calculan a partir de todas las dimensiones del dataset multidimensional de entrada.

Nombre de tabla de resultados

El nombre de la capa que se creará en Mi contenido y que se agregará al mapa. El nombre predeterminado se basa en el nombre de la herramienta y en el nombre de la capa de entrada. Si la capa ya existe, se le pedirá que indique otro nombre.
