Muestra

Esta herramienta crea una tabla o una clase de entidad de punto que muestra los valores de las celdas de un ráster, o conjunto de rásteres, para ubicaciones definidas. Las ubicaciones se definen por celdas, ráster, entidades de polígono, entidades de polilínea o un conjunto de puntos.

Los valores de píxel se extraerán de todos los rásteres de entrada en cada ubicación. Se creará una tabla o una clase de entidad de punto con campos que contienen los valores de píxel para cada ráster de entrada. En caso de que la tabla de ráster de entrada contenga atributos adicionales, estos no se incluirán en la tabla de salida.

Cualquier combinación de rásteres, como por ejemplo de banda única, multibanda, multidimensional o multivariante, se puede especificar como rásteres de entrada. La estructura de la tabla de salida cambia cuando los rásteres de entrada son multidimensionales. Cuando se especifica un ráster multibanda como uno de los rásteres de entrada, se utilizan todas las bandas de esa entrada. Para procesar una selección de bandas a partir de un ráster multibanda de entrada, cree primero un dataset ráster nuevo formado por esas bandas concretas. Utilice el resultado en la lista de rásteres de entrada.

La ubicación de entrada puede ser un ráster o una entidad de punto:

  • Ráster: los píxeles que contienen valores válidos (que no sean NoData) se utilizan para extraer valores de píxel de todos los rásteres de entrada, y el centro del píxel se utiliza para ubicaciones de punto.
  • Punto: valores que se muestrearán en cada ubicación de punto.
  • Polilínea o un polígono: valores que se muestrearán en el centroide de polígono o polilínea. Si el ráster de entrada es multidimensional, los valores se agregarán en función del tipo de Estadísticas.

Los rásteres de entrada se muestrean en su referencia y resolución espaciales nativas. Si hubiera varios rásteres de entrada con diferentes referencias espaciales, las ubicaciones de entrada se proyectan en la referencia espacial de cada ráster individual desde la que se extraen los valores. El entorno de análisis se aplica a las ubicaciones de entrada, aunque los rásteres de entrada no respetarán el entorno de análisis.

A las ubicaciones que extraen valores de celdas NoData en el ráster de entrada se les proporcionará un valor <null> en la tabla de salida.

Para realizar análisis con entidades multipunto, conviértalas primero a entidades de un solo punto antes de usarlas en esta herramienta.

La referencia espacial de las coordenadas x,y de la tabla de muestra de salida es la misma que la de las entidades de puntos o ráster de ubicación de entrada salvo que se especifique en el entorno Sistema de coordenadas de salida. Si no se conoce la referencia espacial de los datos de ubicación de entrada, la referencia espacial de las coordenadas x,y tampoco se conoce.

La referencia espacial de la clase de entidad de salida es la misma que la de las entidades de puntos o ráster de ubicación de entrada, salvo que se especifique en el entorno Sistema de coordenadas de salida.

El entorno Factor de procesamiento en paralelo solo se admite cuando la entrada es un ráster multidimensional.

Si Usar la extensión de mapa actual está activado, se analizarán las variables o bandas de la capa que estén visibles en la extensión de mapa actual. Si no está activado, se analizarán las variables o bandas de la capa completa, incluso si se encuentran fuera de la extensión de mapa actual.

Elegir capas de imágenes para muestreo

Una única capa de imágenes o una lista de capas de imágenes cuyos valores se muestrearán según los datos de ubicación de entrada.

Para muestrear todas las porciones de una capa de imágenes multidimensional, debe ser el único ráster de entrada.

Elegir capa de ubicación

Los datos que identifican las posiciones que desea muestrear.

Los datos de ubicación de entrada pueden ser una clase de entidad de punto, polilínea o polígono.

Especificar campo de Id. único

Un campo que contiene un valor diferente para cada ubicación o entidad en el ráster de ubicación o clase de entidad de entrada.

Especificar información de adquisición de datos de ubicación (opcional)

Especifique la hora, profundidad u otros datos de adquisición asociados con las entidades o celdas de ubicación.

Para cada dimensión, existen tres opciones para especificar la información de adquisición asociada con celdas o entidades de ubicación:

  • Campo o valor único: cuando se selecciona esta opción, debe especificarse la siguiente información:
    • Campo o valor de dimensión: seleccione un campo de valores de dimensión desde los datos de ubicación o introduzca un valor de dimensión específico.
  • Campo o valor único con valores relativos: si se selecciona esta opción, la información siguiente debe especificarse junta para definir el rango en el que se calcularán las estadísticas para variables:
    • Campo o valor de dimensión: seleccione un campo de valores de dimensión desde los datos de ubicación.
    • Valor relativo anterior: especifique un número no negativo utilizado que se restará de Campo o valor de dimensión para definir el inicio de un rango de valores de dimensión. Este valor representa el número de días si la dimensión es tiempo.
    • Valor relativo posterior: especifique un número no negativo utilizado para sumarlo a Campo o valor de dimensión para definir el final de un rango de valores de dimensión. Este valor representa el número de días si la dimensión es tiempo.
  • Campos o valores de inicio y fin: si se selecciona esta opción, la información siguiente debe especificarse junta para definir el rango en el que se calcularán las estadísticas para variables:
    • Campo o valores de inicio: seleccione un campo de valores de dimensión desde los datos de ubicación o introduzca un valor de dimensión específico para definir el inicio de un rango de valores de dimensión.
    • Campo o valores de finalización: seleccione un campo de valores de dimensión desde los datos de ubicación o introduzca un valor de dimensión específico para definir el final de un rango de valores de dimensión.

Especificar campo o valor de distancia de zona de influencia

La distancia especificada que rodea las entidades de datos de ubicación. La distancia de zona de influencia se especifica en las unidades lineales de la referencia espacial de la entidad de ubicación. Si la entidad utiliza una referencia geográfica, la unidad estará en grados.

Se calcularán las estadísticas dentro de esta área de zona de influencia.

Elegir tipo de estadísticas

El tipo de estadística que se calculará.

  • Mínimo: busca el valor mínimo dentro del rango especificado.
  • Máximo: busca el valor máximo dentro del rango especificado.
  • Mediana: busca el valor de la mediana dentro del rango especificado.
  • Valor medio: calcula el promedio del rango especificado.
  • Suma: calcula la suma de las variables dentro del rango especificado.
  • Mayoría: busca el valor que se produce con más frecuencia.
  • Minoría: busca el valor que se produce con menos frecuencia.
  • Desviación estándar: calcula la desviación estándar.
  • Percentil: calcula un percentil definido dentro del rango especificado.

Valor de percentil

El percentil que se va a calcular cuando el parámetro Tipo de estadísticas está establecido en Percentil.

El valor puede variar del 0 al 100. El valor predeterminado es 90.

Especificar técnica de remuestreo

El algoritmo de remuestreo utilizado para muestrear un ráster.

  • Vecino: utiliza el valor de la celda más cercana para asignar un valor a la celda de salida durante el remuestreo. Esta es la opción predeterminada.
  • Interpolación bilinear: determina el nuevo valor de una celda basándose en el promedio de distancia ponderada de los cuatro centros de celdas de entrada más cercanos.
  • Convolución cúbica: determina el nuevo valor de una celda basándose en el ajuste de una curva suave a través de los 16 centros de celdas de entrada más cercanos.

Especificar diseño de salida

Especifica si los valores de muestra aparecen en filas o columnas de la tabla de salida. Esta opción solo está disponible cuando el ráster multidimensional de entrada contiene una variable y una dimensión y cada porción es un ráster de una sola banda.

  • Valores muestreados aparecen en fila: los valores muestreados aparecen en filas separadas en la tabla de salida. Esta es la opción predeterminada.
  • Aparecen valores muestreados en columna: los valores muestreados aparecen en columnas separadas en la tabla de salida. Esta opción solo es válida cuando el ráster multidimensional de entrada contiene una variable y una dimensión y cada porción es un ráster de una sola banda.

Especificar tipo de salida

Especifica si se va a generar una clase de entidad de punto con valores muestreados en su tabla, o solo una tabla con valores muestreados.

  • Tabla: genere una tabla con valores muestreados. Esta es la opción predeterminada.
  • Entidades: genere una clase de entidad de punto con valores muestreados en su tabla. Si las entidades de ubicación de entrada son polilíneas o polígonos, cada punto de la salida es el centroide de cada geometría de las entidades de ubicación de entrada.

Nombre de capa de resultados

El nombre de la capa que se creará en Mi contenido y que se agregará al mapa. El nombre predeterminado se basa en el nombre de la herramienta y en el nombre de la capa de entrada. Si la capa ya existe, se le pedirá que indique otro nombre.

Puede especificar el nombre de una carpeta de Mi contenido en la que se guardará el resultado con el cuadro desplegable Guardar el resultado en.
