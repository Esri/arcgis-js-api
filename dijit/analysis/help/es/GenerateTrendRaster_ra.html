Generar ráster de tendencia

Estima la tendencia de cada píxel a lo largo de una dimensión para una o varias variables de un ráster multidimensional.

Elegir capa de imágenes multidimensional para analizar tendencia

La capa de imágenes de entrada de un ráster multidimensional que se analizará.

Elegir dimensión junto con la tendencia de variable que se analizará

La dimensión a lo largo de la cual se extraerá una tendencia para la variable o variables seleccionadas en el análisis.

Elegir variables para analizar tendencia

Las variables para las que se calcularán tendencias. Si no se especifica ninguna variable, se analizará la primera variable del ráster multidimensional.

Elegir el tipo de línea para ajustar valores de variables a lo largo de una dimensión

Especifica el tipo de línea que se utilizará para ajustar a los valores de píxel a lo largo de una dimensión.

  • Lineal: se ajustarán valores de píxel de variable a lo largo de una línea de tendencia lineal. Esta es la opción predeterminada.
  • Polinómico: se ajustarán valores de píxel de variable a lo largo de una línea de tendencia polinómica de segundo orden.
  • Armónico: se ajustarán valores de píxel de variable a lo largo de una línea de tendencia armónica.
  • Mann-Kendall: los valores de píxel variables se evaluarán con la prueba de tendencia Mann-Kendall.
  • Seasonal-Kendall: los valores de píxel variables se evaluarán con la prueba de tendencia Seasonal-Kendall.

Especificar la unidad de tiempo para la duración de un ciclo estacional

Especifica la unidad de tiempo que se utilizará durante el periodo estacional al realizar la prueba Seasonal-Kendall.

  • Días: la unidad de la duración del periodo estacional es días. Esta es la opción predeterminada.
  • Meses: la unidad de la duración del periodo estacional es meses.

Especificar la longitud de ciclo armónico

La longitud de la variación periódica que se va a modelar. Este parámetro se requiere cuando el tipo de línea de tendencia se establece en Armónico. Por ejemplo, el verdor de las hojas tiene con frecuencia un ciclo fuerte de variación en un solo año, por lo que la longitud de ciclo es 1 año. Los datos de temperatura por hora tienen un ciclo fuerte de variación en un solo día, por lo que la longitud de ciclo es 1 día.

La longitud predeterminada es 1 año para los datos que varían en un ciclo anual.

Este parámetro solo se incluye en el análisis de tendencias cuando la dimensión que se va a analizar es tiempo y el tipo de línea de tendencia se establece en Armónico.

Elegir la unidad de tiempo de longitud de ciclo armónico

Especifica la unidad de tiempo que se utilizará para la longitud de un ciclo armónico.

  • Días: la unidad para la longitud del ciclo armónico es días.
  • Años: la unidad para la longitud del ciclo armónico es años. Esta es la opción predeterminada.

Este parámetro solo se incluye en el análisis de tendencias cuando la dimensión que se va a analizar es tiempo y el tipo de línea de tendencia se establece en Armónico.

Especificar el número de frecuencia del ajuste de tendencia de armónico

La frecuencia que se utilizará en el ajuste de tendencia de armónico. Este parámetro especifica la frecuencia de ciclos en un año.

El valor predeterminado es 1, o un ciclo armónico por año.

Este parámetro solo se incluye en el análisis de tendencias cuando la dimensión que se está analizando es tiempo.

Especificar el número de orden polinómico para el ajuste de tendencia

El número de orden polinómico que se utilizará en el ajuste de tendencia polinómica.

Este parámetro solo se incluye en el análisis de tendencias cuando la dimensión que se está analizando es tiempo.

El valor predeterminado es 2, o polinómico de segundo orden.

Elegir estadísticas de modelo que se incluirán en ráster de tendencia

Esta herramienta proporciona tres tipos de estadísticas de la línea ajustada de tendencia: el error cuadrático medio (RMSE), la estadística de idoneidad de ajuste R (R cuadrado) y la estadística de valor P para el coeficiente de pendiente (valor P del coeficiente de pendiente). Cada estadística tiene una casilla de verificación correspondiente para especificar si se calculará y se incluirá como una de las varias bandas en el ráster de tendencia de salida.

  • RMSE

    • Activado: el RMSE se calculará e incluirá como una de las varias bandas en el ráster de tendencia de salida. Esta es la opción predeterminada.
    • Desactivado: el RMSE no se calculará.

  • R cuadrado

    • Activado: el valor de R cuadrado se calculará y se incluirá como una de las varias bandas en el ráster de tendencia de salida.
    • Desactivado: el valor de R cuadrado no se calculará. Esta es la opción predeterminada.

  • Valor P del coeficiente de pendiente

    • Activado: el valor P se calculará e incluirá como una de las varias bandas en el ráster de tendencia de salida.
    • Desactivado: el valor P no se calculará. Esta es la opción predeterminada.

Ignorar valores que faltan en cálculo

Especifica si los valores que faltan se ignorarán en el análisis.

  • Activado: el análisis incluirá todos los píxeles válidos a lo largo de una dimensión determinada y se ignorarán los píxeles NoData. Esta es la opción predeterminada.
  • Desactivado: el análisis dará como resultado NoData si no hay valores NoData para los píxeles a lo largo de la dimensión determinada.

Nombre de capa de resultados

El nombre de la capa que se creará en Mi contenido y que se agregará al mapa. El nombre predeterminado se basa en el nombre de la herramienta y en el nombre de la capa de entrada. Si la capa ya existe, se le pedirá que indique otro nombre.

Puede especificar el nombre de una carpeta de Mi contenido en la que se guardará el resultado con el cuadro desplegable Guardar el resultado en. Si tiene privilegios para crear capas de imágenes tanto en teselas como dinámicas, también puede especificar qué tipo de capa se va a generar para la salida utilizando el cuadro desplegable Guardar el resultado como.
