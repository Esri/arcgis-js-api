入口網站分析環境設定

Portal for ArcGIS 和 ArcGIS Online 中用於分析的環境設定。

覆寫結果圖層

指定是否覆寫現有的圖層。 勾選後，若內容中存在同名的圖層，則會覆寫該圖層。

  • 勾選 - 若內容中存在同名的圖層，則會覆寫該圖層。 輸出將具有輸入的相同座標系統。
  • 取消勾選 - 如果提供的結果圖層名稱已存在，則會發生錯誤，且必須先重命名結果圖層才能執行工具。 這是預設值。

執行分析後關閉分析 widget

指定工具正在處理時，是否要維持開啟和啟用分析對話方塊。 此設定能讓您修改分析參數，同時讓先前的分析工作繼續執行。 分析是一個反覆的程序，且藉由維持工具對話方塊開啟，能讓您變更參數並重新執行分析。

  • 勾選 - 分析工具對話方塊將關閉，且焦點將回到目錄。 這是預設值
  • 取消勾選 - 分析工具對話方塊將維持開啟和啟用。

將分析結果儲存為託管圖徵圖層

指定透過分析建立的結果圖層為圖徵集合或託管圖徵圖層。

  • 勾選 - 分析結果將建立託管圖徵圖層，並在內容中建立項目。 這是預設值。
  • 取消勾選 - 分析結果將建立圖徵集合。

輸出座標系統

指定用於點陣分析之結果圖層的座標系統。

  • 與輸入相同 - 分析結果將位於輸入所在的相同座標系統中。 這是預設值。
  • 依照指定 - 分析結果將位於您選擇的座標系統。 選擇此選項後按一下地球按鈕，並從已知座標系統清單中進行選擇，或在提供的方塊中新增空間參考 WKID。
  • 圖層 <name> - 分析的結果將與您在 Web 地圖中選擇的現有圖層位於相同的座標系統中。

正在處理座標系統

指定您的分析在執行 GeoAnalytics 分析期間將使用的座標系統。

對於 GeoAnalytics，部分分析需要在投影的座標系統中進行處理。 已實作資料或線性單位分級的任何工具都需要投影的座標系統。

  • 與輸入相同 - 將在輸入所在的相同座標系統中處理分析。 這是預設值。
  • 依照指定 - 您的分析將位於您選擇的座標系統。 選擇此選項後按一下地球按鈕，並從已知的投影座標系統清單中進行選擇，或在提供的方塊中新增空間參考 WKID。
  • 圖層 <name> - 您的分析將與您在 Web 地圖中選擇的現有圖層位於相同的座標系統中。

範圍

指定在執行分析期間將使用的範圍或邊界。 將在分析中使用完全位於指定範圍內的所有輸入圖徵，或與指定範圍相交的所有輸入圖徵。

  • 預設值 - 工具提供的範圍。
  • 依照指定 - 使用您提供的座標來定義範圍。
  • 圖層 <name> - 用來處理分析的範圍，將與您在 Web 地圖中選擇的現有圖層的空間範圍相同。

貼齊點陣

調整輸出點陣圖層的範圍，使其在點陣分析中符合指定的 對齊點陣圖層的儲存格對齊。

儲存格大小

指定將在點陣分析中用來建立輸出點陣圖層的儲存格大小或解析度。 預設的輸出解析度由輸入點陣圖層的最大儲存格大小所決定。

  • 輸入的最小值 - 使用所有輸入圖層的最小儲存格大小。
  • 輸入的最大值 - 使用所有輸入圖層的最大儲存格大小。 這是預設值。
  • 依照指定 - 指定用來設定儲存格大小的數值。 如果選擇此設定，則預設值為 1。
  • 圖層 <name> - 將儲存格大小設定為選擇的點陣圖層。

遮罩

指定在點陣分析中將用來定義感興趣區域的圖層。 分析操作只會考量分析遮罩中的儲存格。

  • 遮罩可為點陣或圖徵圖層。
  • 如果分析遮罩是點陣，則會考量具有值的所有儲存格以定義遮罩。 會將遮罩點陣中的 NoData 儲存格視為位於遮罩外部，並且在分析結果圖層中將是 NoData。
  • 如果分析遮罩是圖徵圖層，會在內部將其轉換成點陣加以執行。 因此，請務必針對分析適當地設定 儲存格大小和 對齊點陣 。

資料儲存庫

GeoAnalytics 結果可儲存於 ArcGIS Data Store 並在 Portal for ArcGIS 中顯示為圖徵圖層，或配置的大數據檔案分享。

在 ArcGIS Data Store 中儲存結果時，大多數情況下必須在時空資料儲存庫中儲存結果。 這是預設值。 在下列情況下，您可能想要在關聯式資料儲存庫中儲存結果：

  • 在入口網站之間的協同作業中使用您的結果。
  • 啟用與您結果的同步功能

如果您預期 GeoAnalytics 結果會增加，並想要利用大型的時空資料儲存庫的功能來處理大量資料，則不應使用關聯式資料儲存庫。

平行處理因子

跨多個點陣處理服務執行個體執行分散式分析。

您可以使用支援 平行處理因子環境的工具，控制可用於處理資料的點陣處理服務執行個體的數量。

如果工具不支援 處理器類型，或將 處理器類型環境設為 CPU，則 平行處理因子環境將控制點陣處理 (CPU) 服務執行個體。 若將 處理器類型設為 GPU， 平行處理因子環境將控制處理 GPU 執行個體的點陣數量。

藉由設定 平行處理因子，您可以請求點陣分析圖片伺服器用來處理一個點陣分析任務的平行工作者數量。 不過，如果平行處理的總數超過處理（CPU 或 GPU）服務執行個體的點陣數上限，則會將其他平行處理排入佇列。

若未指定 平行處理因子（預設值），工具將使用處理服務執行個體的點陣數上限的 80%。 可將整數或百分比指定為平行處理因子。

失敗的重試次數

定義在處理特定工作發生隨機失敗時，相同的工作者程序的重試次數。 預設值是 0。

處理工作者的循環利用間隔

定義在重新啟動工作者程序之前將處理多少個影像部分，以防止長時間執行程序所可能發生的失敗情況。 預設值是 0。

處理器類型

您可以使用支援 處理器類型環境的工具來選擇處理資料的位置和方式。 您可以使用伺服器電腦的 CPU 或 GPU 來處理資料。 如果 處理器類型環境空白，工具會使用 CPU 來處理資料。 這是預設值。

可跨多個核心和執行個體進行 CPU 平行處理，這個動作由 平行處理因子所控制。

GPU 可有效地處理圖形和圖片，其高度平行的結構可讓它們以重複且有效的方式處理大資料區塊。 支援此環境的點陣分析工具可跨越多部點陣分析伺服器電腦的 GPU 執行個體分散其工作，這個動作由 平行處理因子所處理。

  • GPU - 將使用圖形處理單元處理資料。
  • CPU - 將使用電腦的中央處理單元處理資料。

重新取樣方法

遵循 重新取樣方法環境的工具會在轉換點陣資料集時插補像素值。 當輸入和輸出不相符、像素大小變更、資料偏移，或上述情況綜合發生時，就會使用此方法。

  • 最近鄰域 - 主要使用於離散資料 (例如土地利用分類)，因為它不會建立新的像素值。 若您想要保留影像中的原始反射率值以進行精確的多光譜分析，此方法也適用於連續資料。 以處理時間來說，這是最有效率的方法，但可能在輸出影像中產生少數位置錯誤。 輸出影像可能出現最多半個像速的偏移，其可能造成影像不連續和外觀呈現鋸齒狀。
  • 雙線性插值 - 此方法最適合用於連續資料。 此方法會執行雙線性插值，並根據四個最近的輸入儲存格中心的加權距離平均值，判斷儲存格的新值。 此外，其建立的輸出影像外觀比「最近鄰域」更平滑，但會修改反射率值，導致影像解析度模糊或喪失。
  • 立方捲繞 - 適用於連續資料。 此方法會執行立方捲繞，並透過 16 個最近輸入儲存格中心來擬合平滑曲線，從而決定儲存格的新值。 其結果的幾何失真比 最近鄰域產生的點陣少，且比 雙線性插值更清晰。 在某些情況下，此方法可能導致輸出像素值超出輸入儲存格值的範圍。 如果這是無法接受的結果，可改用 雙線性插值方法。 立方捲繞需要密集運算，處理時間也較長。
