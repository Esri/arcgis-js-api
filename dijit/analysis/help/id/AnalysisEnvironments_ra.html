Pengaturan Lingkungan Analisis Portal

Pengaturan lingkungan untuk analisis dalam Portal for ArcGIS dan ArcGIS Online.

Timpa layer hasil

Menentukan apakah harus menimpa layer yang sudah ada. Saat dicentang, jika suatu layer dengan nama sama telah ada pada konten Anda, layer itu akan ditimpa.

  • Dicentang—jika suatu layer dengan nama sama telah ada pada konten Anda, layer itu akan ditimpa. Output akan memiliki sistem koordinat yang sama dengan inputnya.
  • Tidak dicentang—Jika nama layer hasil yang diberikan sudah ada, sebuah kesalahan akan terjadi dan layer hasil harus diganti namanya sebelum alat dapat dieksekusi. Ini adalah default.

Tutup widget analisis setelah menjalankan analisis

Menentukan apakah menjaga kotak dialog analisis tetap terbuka dan aktif sementara alat melakukan pemrosesan. Pengaturan ini memungkinkan Anda melakukan modifikasi pada parameter analisis Anda saat tugas analisis sebelumnya masih berjalan. Analisis adalah proses yang berulang, dan dengan menjaga kotak dialog alat tetap terbuka, ini akan memungkinkan Anda melakukan pengubahan parameter dan menjalankan kembali analisis Anda.

  • Dicentang—Kotak dialog alat analisis akan tertutup, dan fokus akan kembali ke daftar isi. Ini adalah default
  • Tidak dicentang—Kotak dialog alat analisis akan tetap terbuka dan aktif.

Simpan hasil analisis sebagai feature layer yang di-host

Menentukan apakah layer hasil yang dibuat dari analisis Anda akan menjadi kumpulan fitur atau feature layer yang di-host.

  • Dicentang—Hasil analisis Anda akan dibuat sebagai feature layer yang di-host, dan sebuah item akan dibuat dalam konten Anda. Ini adalah default.
  • Tidak Dicentang—Hasil analisis Anda akan dibuat sebagai kumpulan fitur.

Sistem koordinat output

Menentukan sistem koordinat layer hasil untuk analisis raster.

  • Sama seperti input—Hasil analisis Anda akan berada dalam sistem koordinat yang sama dengan input. Ini adalah default.
  • Seperti yang ditentukan—Hasil analisis Anda akan berada dalam sistem koordinat yang Anda pilih. Saat opsi ini dipilih, klik tombol bola dunia, lalu pilih dari daftar sistem koordinat yang diketahui, atau tambahkan referensi spasial WKID dalam kotak yang disediakan.
  • Layer <name>—Hasil analisis Anda akan berada dalam sistem koordinat yang sama dengan layer pilihan dalam web map Anda yang sudah ada.

Memproses sistem koordinat

Menentukan sistem koordinat analisis Anda akan digunakan selama eksekusi analisis GeoAnalytics.

Untuk GeoAnalytics, beberapa analisis mengharuskan bahwa proses dieksekusi pada sistem koordinat terproyeksi. Setiap alat yang menerapkan penempatan data atau unit linear mengharuskan sistem koordinat terproyeksi.

  • Sama seperti input—Analisis Anda akan diproses pada sistem koordinat yang sama seperti input. Ini adalah default.
  • Seperti yang ditentukan—Analisis Anda akan berada dalam sistem koordinat yang Anda pilih. Saat opsi ini dipilih, klik tombol bola dunia, lalu pilih dari daftar sistem koordinat terproyeksi yang diketahui, atau tambahkan referensi spasial WKID dalam kotak yang disediakan.
  • Layer <name>— Analisis Anda akan berada dalam sistem koordinat yang sama dengan layer pilihan yang ada dalam web map Anda.

Jangkauan

Menentukan jangkauan atau batasan yang akan digunakan selama eksekusi analisis. Seluruh fitur input yang sepenuhnya berada di dalam atau yang berpotongan dengan jangkauan tertentu akan digunakan pada analisis.

  • Default—Jangkauan yang disediakan oleh alat.
  • Seperti yang ditentukan—Jangkauan ditentukan oleh koordinat yang Anda sediakan.
  • Layer <name>—Jangkauan yang digunakan untuk memproses analisis Anda akan sama dengan jangkauan spasial layer pilihan dalam web map Anda yang ada.

Snap Raster

Sesuaikan jangkauan layer raster output, sehingga akan cocok dengan penyejajaran sel layer Snap Raster yang ditentukan dalam analisis raster.

Ukuran Sel

Menentukan ukuran sel atau resolusi yang akan digunakan untuk membuat layer raster output dalam analisis raster. Resolusi output default ditentukan oleh ukuran sel terbesar dari layer raster input.

  • Input Minimum—Gunakan ukuran sel terkecil dari seluruh layer input.
  • Input Maksimum—Gunakan ukuran sel terbesar dari seluruh layer input. Ini adalah default.
  • Seperti yang ditentukan—Tentukan nilai angka untuk menetapkan ukuran sel. Jika ini dipilih, nilai default adalah 1.
  • Layer <name>—Set the cell size to the raster layer chosen.

Selubung

Menentukan layer yang akan digunakan untuk menentukan area perhatian dalam analisis raster Anda. Hanya sel yang berada dalam mask analisis yang akan diperhitungkan dalam operasi analisis.

  • Mask dapat berupa raster atau feature layer.
  • Jika mask analisis adalah raster, seluruh sel yang memiliki nilai akan diperhitungkan untuk menentukan mask. Sel pada raster mask yang merupakan NoData akan dianggap berada di luar masker dan akan menjadi NoData pada layer hasil analisis.
  • Jika masker analisis adalah feature layer, maka akan diubah secara internal menjadi raster saat eksekusi. Karena alasan ini, pastikan bahwa Ukuran Sel dan Snap Raster diatur dengan baik untuk analisis Anda.

Penyimpanan data

Hasil GeoAnalytics dapat disimpan di ArcGIS Data Store dan ditampilkan sebagai feature layer di Portal for ArcGIS atau berbagi file big data yang dikonfigurasi.

Saat Anda menyimpan hasil di ArcGIS Data Store, biasanya hasil akan disimpan dalam penyimpanan data spatiotemporal. Ini adalah default. Berikut adalah alasan mengapa Anda mungkin ingin menyimpan hasil di sebuah penyimpanan data relasional:

  • Untuk menggunakan hasil di kolaborasi portal-ke-portal
  • Untuk mengaktifkan kemampuan sinkronisasi dengan hasil Anda

Anda tidak boleh menggunakan penyimpanan data relasional jika Anda berharap hasil GeoAnalytics dapat meningkat dan ingin memanfaatkan kemampuan penyimpanan big data spatiotemporal untuk menangani jumlah data yang besar.

Faktor pemrosesan paralel

Mendistribusikan analisis pada beberapa contoh layanan Pemrosesan Raster.

Alat yang mematuhi lingkungan Faktor pemrosesan paralel memungkinkan Anda untuk mengendalikan jumlah contoh layanan pemrosesan raster yang dapat digunakan untuk memproses data Anda.

Jika alat tidak mematuhi Jenis prosesor, atau jika lingkungan Jenis prosesor diatur ke CPU, lingkungan Faktor pemrosesan paralel mengendalikan contoh layanan pemrosesan raster (CPU). Jika Jenis prosesor diatur ke GPU, lingkungan Faktor pemrosesan paralel mengendalikan jumlah contoh raster yang memproses GPU.

Dengan mengatur Faktor pemrosesan paralel, Anda dapat meminta jumlah pekerja paralel yang digunakan server gambar analisis raster untuk memproses satu tugas analisis raster. Namun, jika jumlah total pemrosesan paralel melebihi jumlah maksimal pemrosesan contoh layanan pemrosesan raster (CPU atau GPU), proses paralel tambahan akan diantrekan.

Jika Faktor pemrosesan paralel tidak ditentukan, yang mana merupakan default, alat akan menggunakan 80 persen jumlah maksimal untuk raster memproses contoh layanan. Baik bilangan bulat atau persentase dapat ditentukan sebagai faktor pemrosesan paralel.

Jumlah Percobaan yang Gagal

Menentukan berapa banyak percobaan yang akan dilakukan oleh proses pekerja yang sama bila ada kegagalan acak dalam memproses pekerjaan tertentu. Nilai default adalah 0.

Daur Ulang Interval Pekerja Pemrosesan

Tetapkan berapa banyak bagian gambar yang akan diproses sebelum memulai ulang proses pekerja, untuk mencegah kemungkinan kegagalan dalam proses yang berjalan lama. Nilai default adalah 0.

Jenis prosesor

Alat yang mematuhi lingkungan Jenis prosesor memungkinkan Anda untuk memilih tempat dan bagaimana Anda ingin memproses data Anda. Anda dapat memproses data Anda menggunakan CPU atau GPU komputer server. Jika lingkungan Jenis prosesor kosong, alat menggunakan CPU untuk memproses data. Ini adalah default.

Memproses CPU dapat diparalelkan di beberapa inti dan contoh, sebagaimana ditangani oleh Faktor pemrosesan paralel.

GPU bekerja efektif pada pemrosesan grafik dan gambar, yang mana struktur paralel tingginya menjadikannya efisien dalam memproses blok data yang besar secara repetitif. Alat Analisis Raster yang mematuhi lingkungan ini dapat membagikan tugasnya di seluruh contoh GPU pada beberapa mesin server analisis raster, sebagaimana ditangani oleh Faktor pemrosesan paralel.

  • GPU—Data akan diproses menggunakan unit pemrosesan grafik.
  • CPU—Data akan diproses menggunakan unit pemrosesan sentral komputer.

Metode pengambilan sampel ulang

Alat yang sesuai dengan lingkungan Metode pengambilan sampel ulang menginterpolasi nilai piksel sekaligus mengubah set data raster Anda. Ini digunakan ketika input dan output tidak sejajar, ketika ukuran piksel berubah, saat data digeser, atau kombinasi keduanya.

  • Lingkungan terdekat—Digunakan terutama untuk data diskret, seperti klasifikasi penggunaan lahan, karena tidak membuat nilai piksel baru. Metode ini juga sesuai untuk data berkelanjutan jika Anda ingin mempertahankan nilai reflektansi aslinya dalam citra untuk analisis multispektral yang akurat. Metode ini adalah yang paling efisien dari segi waktu pemrosesan, tetapi dapat menyebabkan kesalahan posisi kecil pada gambar output. Gambar output dapat meleset hingga setengah piksel, yang dapat menyebabkan gambar mengalami diskontinuitas dan tampilan bergerigi.
  • Interpolasi bilinear—Metode ini paling sesuai untuk data berkelanjutan. Metode ini melakukan interpolasi bilinear dan menentukan nilai baru sel berdasarkan rata-rata jarak tertimbang dari empat pusat sel input terdekat. Metode ini menciptakan gambar output yang penampilannya lebih halus daripada lingkungan Terdekat tetapi mengubah nilai reflektansi, yang mengakibatkan kabur atau hilangnya resolusi gambar.
  • Konvolusi kubik—Cocok untuk data berkelanjutan. Metode ini melakukan konvolusi kubik dan menentukan nilai baru sel berdasarkan pemasangan kurva halus melalui 16 pusat sel input terdekat. Hasilnya secara geometris kurang terdistorsi daripada raster yang dihasilkan dengan Lingkungan terdekat, dan lebih tajam daripada Interpolasi bilinear. Dalam beberapa kasus, metode ini dapat menghasilkan nilai piksel output di luar rentang nilai sel input. Jika metode ini tidak dapat diterima, gunakan metode interpolasi Bilinear sebagai gantinya. Konvolusi kubik intensif secara komputasi dan membutuhkan waktu lebih lama untuk diproses.
